# 快速参考

> 本文档提供速查表和决策流程图，方便 AI 快速查询和决策

---

## 1. 文档层级速查表

| 层级 | 目录 | 回答的问题 | 必须包含 | 禁止包含 |
|------|------|-----------|---------|---------|
| **L1 系统级** | `01-系统设计/` | 系统做什么、整体技术路线 | 架构图、模块列表、技术选型 | 具体算法代码、参数值 |
| **L2 规范级** | `02-技术规范/` | 跨模块的权威定义 | 数据结构定义、接口签名、约束条件 | 实现代码、模块特定逻辑 |
| **L3 模块级** | `03-模块设计/` | 某个模块如何实现 | 定位、输入输出、算法、测试用例 | 其他模块的实现细节 |
| **L4 验证级** | `04-测试验证/` | 如何验证技术方案 | 测试计划、性能指标、验收标准 | 实现代码（除非是测试工具） |
| **L5 报告级** | `05-报告/` | 某次实验或阶段结果 | 执行摘要、结果、结论 | 待办事项（应放在项目计划） |
| **临时** | `temp/` | 临时文件 | 任何 | 正式文档引用 |

## 2. 决策流程图
### 2.1 新增功能决策
```
用户请求新增功能
  ↓
是否创建新模块？
  ├─ 是 ──→ 是否需要新数据结构？
  │         ├─ 是 ──→ 定义数据结构 → 创建模块文档 → 编码
  │         └─ 否 ──→ 创建模块文档 → 编码
  └─ 否 ──→ 修改现有模块文档 → 编码
```

### 2.2 修改代码决策
```
用户请求修改代码
  ↓
分析变更类型
  ├─ 数据结构变更 ──→ 更新数据结构规范 → 影响分析 → 更新相关文档 → 修改代码
  ├─ 接口变更 ──→ 更新接口规范 → 影响分析 → 更新相关文档 → 修改代码
  ├─ 算法变更 ──→ 更新模块文档 → 修改代码
  └─ 简单修复 ──→ 修改代码 → 更新版本历史
```

### 2.3 Bug 修复决策
```
用户报告 Bug
  ↓
Bug 是否影响接口？
  ├─ 是 ──→ 按接口变更流程处理
  └─ 否 ──→ Bug 是否影响数据结构？
              ├─ 是 ──→ 按数据结构变更流程处理
              └─ 否 ──→ 直接修复 → 更新版本历史
```

## 3. 快速检查清单
### 3.1 编码前检查
```
□ 需求已明确
□ 文档已创建（如需）
  ├─ □ 系统架构设计（如需）
  ├─ □ 数据结构规范（如需）
  ├─ □ 接口规范（如需）
  └─ □ 模块设计文档
□ 文档已完善（核心章节已填充）
□ 版本历史已添加
□ 一致性检查通过（≥ 95%）
```

### 3.2 提交前检查
```
□ 代码与文档一致
□ 单元测试通过
□ 测试用例与文档定义一致
□ 临时文件已清理或迁移
□ 版本历史已更新
```

### 3.3 创建文档检查
```
□ 使用了正确的模板
□ 文档层级正确（L1-L5）
□ 添加了文档定位说明
□ 引用了数据结构/接口（而非复制）
□ 添加了版本历史
```

## 4. 常见场景关键词识别
### 4.1 触发标准流程的关键词
| 场景 | 触发关键词 |
|------|-----------|
| 新模块开发 | "实现新"、"创建新"、"开发新"、"添加新功能"、"新建模块" |
| 数据结构变更 | "改字段"、"修改数据结构"、"变更类型"、"添加字段"、"删除字段" |
| 接口变更 | "改接口"、"修改函数签名"、"添加参数"、"修改返回值"、"改参数" |
| 算法变更 | "优化算法"、"改算法"、"重构逻辑"、"改进实现" |
| 文档编写 | "写文档"、"创建文档"、"补充文档"、"更新设计" |

### 4.2 触发简化流程的关键词
| 场景 | 触发关键词 |
|------|-----------|
| 快速原型 | "poc"、"demo"、"spike"、"原型"、"快速验证"、"试验" |
| 简单修复 | "小修改"、"快速修复"、"简单改动"（且不涉及接口/数据结构） |
| 配置修改 | "改配置"、"修改 README"、"更新 .gitignore" |

### 4.3 不触发流程的关键词
| 场景 | 关键词 |
|------|--------|
| 代码阅读 | "查看"、"阅读"、"分析"、"理解"、"解释" |
| 问题排查 | "排查"、"调查"、"debug"、"查问题" |
| 简单查询 | "什么是"、"说明"、"介绍"、"解释一下" |

## 5. 文档模板速查
### 5.1 模块设计文档必填章节
```markdown
# <模块名>设计

> **文档定位**：本文档属于 L3 模块级，回答"<模块名>如何实现"。

## 1. 模块定位与边界
- 做什么：<功能>
- 不做什么：<非功能>

## 2. 输入输出数据规范
- 输入：引用 [数据结构](链接)
- 输出：引用 [数据结构](链接)

## 3. 核心算法/逻辑设计
（算法流程、关键参数）

## 4. 接口设计
- 引用 [接口规范](链接)

## 5. 验证方案与测试用例
| 用例ID | 输入 | 预期输出 |

## 6. 已知约束与改进路线图
（可选）

## 7. 版本历史
| 版本 | 日期 | 说明 |
```

### 5.2 数据结构规范必填内容
```markdown
### <数据结构名>

**定义**：
```python
class DataStructureName:
    field1: Type1
    field2: Type2
```

**字段说明**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

**约束条件**：
- 约束1

**被以下模块使用**：
- [模块A](链接)
```

### 5.3 接口规范必填内容
```markdown
### <接口名>

**接口签名**：
```python
def function_name(param1: Type1, param2: Type2) -> ReturnType:
    """接口说明"""
```

**参数说明**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

**返回值**：
- 类型：ReturnType
- 说明：<返回值说明>

**异常**：
| 异常类型 | 触发条件 |
```

## 7. 常见错误速查
| 错误 | 正确做法 | 参考 |
|------|---------|------|
| 重复定义数据结构 | 引用数据结构规范 | anti-patterns.md#反模式1 |
| 临时文件放在正式目录 | 放在 temp/ 目录 | anti-patterns.md#反模式2 |
| 版本历史缺少影响范围 | 添加"影响范围：..." | anti-patterns.md#反模式3 |
| 文档与代码不一致 | 以文档为准，或更新文档 | anti-patterns.md#反模式5 |
| 缺少文档定位说明 | 添加"文档定位"章节 | anti-patterns.md#反模式6 |
| 模块文档记录工作进展 | 只在项目计划中记录 | anti-patterns.md#反模式7 |
| 使用裸字典传数据 | 使用规范数据结构 | anti-patterns.md#反模式8 |
| 代码中重复文档内容 | 代码注释引用文档 | anti-patterns.md#反模式8 |
| 缺少 README 导航 | 创建 doc/README.md | anti-patterns.md#反模式9 |
| 忘记更新关联文档 | 执行影响分析 | anti-patterns.md#反模式10 |

## 8. AI 执行决策树
```
收到用户请求
  ↓
识别请求类型
  ├─ 新增功能 ──→ 使用"新增功能决策流程"
  ├─ 修改代码 ──→ 使用"修改代码决策流程"
  ├─ Bug 修复 ──→ 使用"Bug 修复决策流程"
  ├─ 快速原型 ──→ 使用 POC 流程
  ├─ 文档编写 ──→ 检查是否涉及代码
  │              ├─ 是 ──→ 使用迭代变更流程
  │              └─ 否 ──→ 直接创建/更新文档
  └─ 查询/分析 ──→ 不触发 dev-workflow
```
