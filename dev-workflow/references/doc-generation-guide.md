## 1. 从代码生成文档

### 1.1 适用场景
- 代码已存在，但缺少文档
- 需要为现有代码补充文档
- 重构后需要更新文档

### 1.2 生成流程

```python
def 从代码生成文档(代码文件):
    """从代码生成文档的流程"""

    print(f"📝 从代码生成文档：{代码文件}")

    # Step 1: 分析代码结构
    print("\nStep 1: 分析代码结构...")
    代码分析 = {
        "类定义": 提取类定义(代码文件),
        "函数签名": 提取函数签名(代码文件),
        "依赖关系": 分析依赖(代码文件),
        "数据流": 分析数据流(代码文件)
    }

    # Step 2: 确定文档类型
    print("\nStep 2: 确定文档类型...")
    if 是单个模块(代码分析):
        文档类型 = "模块设计文档"
        文档路径 = f"doc/03-模块设计/{提取模块名(代码文件)}.md"
    elif 是整体架构(代码分析):
        文档类型 = "系统架构设计"
        文档路径 = "doc/01-系统设计/系统架构设计.md"
    else:
        文档类型 = "模块设计文档"
        文档路径 = "doc/03-模块设计/未命名模块.md"

    print(f"  文档类型：{文档类型}")
    print(f"  文档路径：{文档路径}")

    # Step 3: 检查文档是否已存在
    if exists(文档路径):
        print(f"\n⚠️  文档已存在：{文档路径}")
        print("  建议：更新现有文档而非重新生成")
        return "更新现有文档"

    # Step 4: 选择模板
    print("\nStep 4: 选择模板...")
    模板 = 查找模板(文档类型)
    if not 模板:
        print("  ⚠️  未找到模板，将使用默认结构")
        模板 = 默认模板()

    # Step 5: 填充内容
    print("\nStep 5: 填充内容...")
    填充内容 = {
        "模块名": 提取模块名(代码文件),
        "文件结构": 提取文件结构(代码文件),
        "接口": 代码分析["函数签名"],
        "依赖": 代码分析["依赖关系"]
    }

    # Step 6: 生成文档框架
    print("\nStep 6: 生成文档框架...")
    文档内容 = 填充模板(模板, 填充内容)

    # Step 7: 识别需要补充的部分
    print("\nStep 7: 识别需要补充的部分...")
    需要补充 = []
    if not 代码分析.get("核心算法"):
        需要补充.append("核心算法/逻辑设计")
    if not 代码分析.get("测试用例"):
        需要补充.append("验证方案与测试用例")
    if not 代码分析.get("设计决策"):
        需要补充.append("设计决策及依据")

    # 输出结果
    print("\n✅ 文档框架已生成")
    print(f"  文档路径：{文档路径}")
    if 需要补充:
        print("\n  以下章节需要你补充：")
        for 章节 in 需要补充:
            print(f"  - {章节}")

    return 文档路径
```

## 2. 从需求生成文档

### 2.1 适用场景
- 新功能开发
- 从零开始设计
- 需求分析阶段

### 2.2 生成流程

```python
def 从需求生成文档(需求描述):
    """从需求描述生成文档"""

    print(f"📝 从需求生成文档")

    # Step 1: 提取关键信息
    print("\nStep 1: 提取关键信息...")
    关键信息 = {
        "功能描述": 提取功能描述(需求描述),
        "输入输出": 推断输入输出(需求描述),
        "约束条件": 提取约束条件(需求描述),
        "性能要求": 提取性能要求(需求描述)
    }

    # Step 2: 推断数据结构需求
    print("\nStep 2: 推断数据结构需求...")
    数据结构需求 = []
    for io in 关键信息["输入输出"]:
        数据结构 = {
            "名称": io["名称"],
            "字段": 推断字段,
            "约束": io["约束"]
        }
        数据结构需求.append(数据结构)

    # Step 3: 确定文档类型和路径
    print("\nStep 3: 确定文档类型...")
    if "模块" in 需求描述 or "功能" in 需求描述:
        文档类型 = "模块设计文档"
        文档路径 = f"doc/03-模块设计/{推断模块名(需求描述)}.md"
    else:
        文档类型 = "需求文档"
        文档路径 = f"doc/00-需求/{推断需求名(需求描述)}.md"

    # Step 4: 生成文档框架
    print("\nStep 4: 生成文档框架...")
    文档框架 = 生成文档框架(文档类型, 关键信息, 数据结构需求)

    # Step 5: 识别缺失信息
    print("\nStep 5: 识别缺失信息...")
    缺失信息 = []
    if not 关键信息.get("输入输出的具体格式"):
        缺失信息.append("输入输出数据的具体格式")
    if not 关键信息.get("算法要求"):
        缺失信息.append("算法/逻辑设计")
    if not 关键信息.get("测试标准"):
        缺失信息.append("验收标准和测试用例")

    # 输出结果
    print("\n✅ 文档框架已生成")
    print(f"  文档路径：{文档路径}")
    if 缺失信息:
        print("\n  以下信息需要你提供：")
        for 信息 in 缺失信息:
            print(f"  - {信息}")

    return 文档路径
```

## 3. 交互式文档生成

### 3.1 逐章节生成
AI 应该采用交互式方式，逐章节询问用户：

```python
def 交互式生成文档(文档类型):
    """交互式生成文档"""

    print(f"🤝 交互式生成{文档类型}")

    # Step 1: 基础信息
    模块名 = input("模块名称：")
    功能描述 = input("功能描述：")

    # Step 2: 输入输出
    print("\n请描述输入数据：")
    输入 = []
    while True:
        输入项 = input("  输入项（输入 q 结束）：")
        if 输入项 == 'q':
            break
        输入.append(输入项)

    print("\n请描述输出数据：")
    输出 = []
    while True:
        输出项 = input("  输出项（输入 q 结束）：")
        if 输出项 == 'q':
            break
        输出.append(输出项)

    # Step 3: 核心算法
    print("\n核心算法/逻辑设计：")
    print("  1. 你提供描述")
    print("  2. 跳过（稍后补充）")
    选择 = input("选择：")
    if 选择 == "1":
        算法描述 = input("算法描述：")
    else:
        算法描述 = "（待补充）"

    # Step 4: 生成文档
    文档内容 = 组装文档(模块名, 功能描述, 输入, 输出, 算法描述)

    return 文档内容
```

## 4. 文档生成最佳实践

### 4.1 应该做的
✅ **提供框架而非完整内容**
- 生成结构化框架
- 标记需要补充的部分
- 提供填写建议

✅ **保留人工判断**
- 不自动决定算法选择
- 不自动定义约束条件
- 不自动生成测试用例细节

✅ **引用而非创建**
- 检查数据结构是否已定义
- 检查接口是否已定义
- 建议引用现有规范

✅ **交互式生成**
- 逐章节询问
- 允许跳过
- 允许稍后补充

### 4.2 不应该做的
❌ **不要完全自动化**
- 文档包含设计决策，需要人工判断
- 不要试图"一次生成完整文档"

❌ **不要猜测设计决策**
- 不确定时应该询问用户
- 不要自己编造算法或约束

❌ **不要忽略文档规范**
- 必须包含版本历史
- 必须引用数据结构规范
- 必须有文档定位说明

❌ **不要覆盖现有文档**
- 文档已存在时应该更新而非重新生成
- 询问用户是更新还是备份后重新生成

## 5. 快速参考表

| 场景 | 生成方式 | AI 角色 | 用户角色 |
|------|---------|---------|---------|
| 代码已存在 | 从代码生成 | 分析代码结构、生成框架 | 补充算法说明、测试用例 |
| 需求已明确 | 从需求生成 | 提取关键信息、推断数据结构 | 确认数据结构、补充算法 |
| 全新项目 | 交互式生成 | 引导填写、组装文档 | 提供信息、做出决策 |
| 文档已存在 | 更新现有文档 | 分析差异、建议更新内容 | 决定更新策略 |
| 文档不完整 | 补充缺失章节 | 识别缺失部分、提供模板 | 填充内容 |
