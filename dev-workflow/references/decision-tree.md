# 场景决策树

> 本文档提供常见开发场景的决策流程，帮助 AI 快速判断应该如何执行

---

## 场景 1：新增功能

### 决策流程

```
开始
  ↓
是否创建新模块？ ── 否 ──→ 修改现有模块 → 转场景 2
  │
 是
  ↓
是否需要新的数据结构？ ── 否 ──→ 检查是否需要新的接口？
  │                                      │
 是                                    否
  ↓                                    ↓
更新数据结构规范                  检查是否需要新的接口？
  │                                    │
 是                                    否
  ↓                                    ↓
检查是否需要新的接口？            创建模块设计文档
  │                                    │
 否                                    ↓
 是                                 ┌──────────────┐
  ↓                                 │            │
创建模块设计文档                  │            │
  ↓                                 │            ↓
开始编码实现                            开始编码实现
```

### AI 行动指南

1. **检测到新模块开发**：
   - 输出：`⚠️ 检测到新模块开发，需要先创建设计文档`
   - 执行：跳转到"文档创建流程"

2. **检测到新数据结构需求**：
   - 输出：`📝 需要在数据结构规范中定义：{数据结构名称}`
   - 执行：询问用户是否现在定义数据结构

3. **文档已完备**：
   - 输出：`✅ 文档已完备，开始编码实现`
   - 执行：开始编写代码

## 场景 2：修改现有代码

### 决策流程

```
开始
  ↓
分析变更类型
  ├─ 数据结构变更 ──→ 执行"迭代变更流程（数据结构变更）"
  ├─ 接口变更 ──→ 执行"迭代变更流程（接口变更）"
  ├─ 算法变更 ──→ 更新模块设计文档版本历史 → 修改代码
  ├─ 参数调整 ──→ 更新模块设计文档版本历史 → 修改代码
  └─ 简单修复 ──→ 直接修改代码 → 更新版本历史
```

### AI 行动指南

1. **检测到数据结构变更**：
   - 输出：`📋 影响分析：需要更新数据结构规范和相关文档`
   - 执行：跳转到"迭代变更流程"

2. **检测到接口变更**：
   - 输出：`📋 影响分析：需要更新接口规范和相关文档`
   - 执行：跳转到"迭代变更流程"

3. **简单修复（不影响接口）**：
   - 输出：`✅ 磀测到简单修复，直接修改代码`
   - 执行：修改代码并更新版本历史

## 场景 3： Bug 修复

### 决策流程

```
开始
  ↓
Bug 是否涉及接口变更？
  ├─ 是 ──→ 执行"迭代变更流程（接口变更）"
  └─ 否 ──→ Bug 是否涉及数据结构变更？
              ├─ 是 ──→ 执行"迭代变更流程（数据结构变更）"
              └─ 否 ──→ 直接修复 Bug → 更新版本历史
```

### AI 行动指南

1. **Bug 修复涉及接口**：
   - 输出：`⚠️ Bug 修复涉及接口变更，需要更新接口规范`
   - 执行：跳转到"迭代变更流程"

2. **Bug 修复不涉及接口**：
   - 输出：`✅ 直接修复 Bug`
   - 执行：修改代码、运行测试、更新版本历史

## 场景 4： 性能优化

### 决策流程

```
开始
  ↓
优化是否改变输入输出？
  ├─ 是 ──→ 执行"迭代变更流程"
  └─ 否 ──→ 优化是否改变核心算法？
              ├─ 是 ──→ 更新模块设计文档（算法章节）→ 执行优化
              └─ 否 ──→ 执行优化 → 更新性能测试报告
```

### AI 行动指南

1. **优化改变输入输出**：
   - 输出：`⚠️ 性能优化改变了输入输出，需要按迭代变更流程执行`
   - 执行：跳转到"迭代变更流程"

2. **优化仅改变算法**：
   - 输出：`📝 需要更新模块设计文档的算法章节`
   - 执行：更新文档、执行优化、更新测试报告

3. **优化不影响接口和算法**：
   - 输出：`✅ 执行性能优化`
   - 执行：优化代码、测试、更新报告

## 场景 5: 重构

### 决策流程

```
开始
  ↓
重构是否改变接口？
  ├─ 是 ──→ 执行"迭代变更流程（接口变更）"
  └─ 否 ──→ 重构是否改变数据结构？
              ├─ 是 ──→ 执行"迭代变更流程（数据结构变更）"
              └─ 否 ──→ 更新模块设计文档（重构说明）→ 执行重构
```

### AI 行动指南

1. **重构改变接口**：
   - 输出：`⚠️ 重构改变了接口，需要更新接口规范`
   - 执行：跳转到"迭代变更流程"

2. **重构不改变接口**：
   - 输出：`📝 需要在模块设计文档中记录重构说明`
   - 执行：更新文档、执行重构、运行测试

## 场景 6: 快速原型（POC）

### 决策流程

```
开始
  ↓
用户是否明确说明是 POC？
  ├─ 是 ──→ 使用 POC 流程
  │           ├─ 创建 temp/spike/ 目录
  │           ├─ 可选：在 temp/drafts/ 记录思路
  │           └─ 执行 POC 开发
  └─ 否 ──→ 使用标准流程
```

### AI 行动指南

1. **用户明确 POC**：
   - 输出：`🚀 检测到快速原型请求，使用简化流程`
   - 执行：在 temp/ 目录下创建 POC，不强制要求完整文档

2. **用户未说明 POC**：
   - 输出：`将按照标准开发流程进行`
   - 执行：使用标准文档驱动流程

## 场景 7: 文档更新

### 决策流程

```
开始
  ↓
文档更新类型？
  ├─ 补充缺失章节 ──→ 直接更新文档
  ├─ 修改已有内容 ──→ 是否涉及代码变更？
  │                          ├─ 是 ──→ 执行"迭代变更流程"
  │                          └─ 否 ──→ 更新版本历史
  └─ 生成新文档 ──→ 使用模板创建 → 填充内容
```

### AI 行动指南

1. **补充文档**：
   - 输出：`📝 补充文档缺失章节`
   - 执行：读取文档、识别缺失章节、补充内容

2. **修改文档内容**：
   - 输出：`📝 修改文档内容`
   - 执行：判断是否影响代码、更新文档、更新版本历史

3. **生成新文档**：
   - 输出：`📝 创建新文档`
   - 执行：选择模板、填充内容、添加版本历史

## 快速决策表

| 场景 | 关键判断 | 执行流程 |
|------|---------|---------|
| 新增功能 | 是否新模块？ | 新模块：创建文档 → 编码<br>现有模块：修改文档 → 编码 |
| 修改代码 | 是否改变接口/数据结构？ | 是：迭代变更流程<br>否：直接修改 |
| Bug 修复 | 是否影响接口？ | 是：迭代变更流程<br>否：直接修复 |
| 性能优化 | 是否改变输入输出？ | 是：迭代变更流程<br>否：直接优化 |
| 重构 | 是否改变接口？ | 是：迭代变更流程<br>否：更新文档 → 重构 |
| POC | 用户是否明确说明？ | 是：POC 流程<br>否：标准流程 |
| 文档更新 | 是否涉及代码？ | 是：迭代变更流程<br>否：直接更新 |
