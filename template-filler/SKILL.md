# template-filler - 对话式模板文档填写

> 读取用户提供的模板，分段对话收集信息，逐段展示草稿确认，最终保存完整文档文件。

## 核心流程（5步）

| 步骤 | 动作 | AI 行为 |
|------|------|---------|
| 1 | 读取模板 | 读取用户指定的模板文件路径 |
| 2 | 分析结构 | 识别所有 Markdown 章节（H2/H3），制定问题清单 |
| 3 | 分段访谈 | 逐章节：提问（1-3 个）→ 收集回答 |
| 4 | 分段预览 | 展示该章节草稿 → 用户确认或修改 |
| 5 | 保存文件 | 所有章节完成后，生成完整文档并写入文件 |

## 访谈协议（AI 须遵守）

- 每次只处理一个章节
- 每个章节提问不超过 3 个（复杂章节可追问）
- 每个章节草稿展示后等待确认，"OK/好/继续" 才进入下一章节
- 用户说"跳过"则跳过该章节（保留模板原文或留空）
- 用户说"返回"则重新访谈上一个章节
- 所有章节完成后，询问保存路径（默认：模板同目录，文件名去掉 .template 后缀或加 _filled）

## 决策树（进行中遇到问题时）

**用户回答模糊？**
- YES → 追问澄清，给出选项帮助用户选择
- NO  → 直接用回答生成草稿

**用户对草稿不满意？**
- 局部修改 → 用户指出哪里不对，AI 修改后再展示
- 重来 → 重新提问该章节

## 典型使用场景

```
用户: 帮我按这个模板写文档 /path/to/template.md
AI: 已读取模板，共 5 个章节：[列出章节名]
    我们从第一章节「项目背景」开始。
    问题1：这个项目要解决什么核心问题？

用户: 是一个...
AI: [展示「项目背景」草稿]
    ---
    ## 项目背景
    （生成的内容）
    ---
    这样描述准确吗？说"继续"进入下一章节，或告诉我需要修改哪里。
```

## 常见问题

| 情况 | 处理方式 |
|------|---------|
| 模板路径不存在 | 提示用户检查路径并重试 |
| 模板无 Markdown 标题 | 将整个文件视为一个章节 |
| 用户想一次提供所有信息 | 接受，直接生成草稿供确认 |
| 章节内有子章节（H3） | 将 H3 视为 H2 的子问题，合并访谈 |
