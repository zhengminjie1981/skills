# 场景决策树

本文档提供 AI Agent 在处理知识库索引相关请求时的决策流程。

## 决策流程总览

```
用户请求 → 识别意图 → 选择操作 → 执行 → 验证 → 输出
```

## 场景 1: 建立索引

### 决策树

```
用户请求建立索引
    │
    ├─ 关键词识别
    │   • "建立索引"、"创建索引"、"为...建立索引"
    │   • "帮我索引这个文件夹"
    │   • "构建知识库"
    │
    ├─ 路径识别
    │   ├─ 明确指定路径 → 使用指定路径
    │   ├─ 模糊指定（如"信息化"）→ 搜索匹配路径
    │   └─ 未指定 → 询问用户
    │
    ├─ 检查现有索引
    │   ├─ 索引已存在
    │   │   └─ 询问: "索引已存在，是否重建或更新？"
    │   │       ├─ 重建 → 删除旧索引，重新构建
    │   │       └─ 更新 → 执行增量更新
    │   └─ 索引不存在 → 直接构建
    │
    ├─ 执行构建流程
    │   ├─ 扫描文档
    │   ├─ 检测工具
    │   ├─ 转换文档
    │   ├─ 生成摘要
    │   └─ 写入索引
    │
    └─ 输出构建报告
```

### AI 行动指南

**检测到用户请求建立索引时**:

1. **输出确认信息**:
   ```
   ✓ 收到请求：为「[知识库名]」建立索引
   ```

2. **检查现有索引**:
   - 如果索引已存在，输出:
     ```
     ⚠️ 检测到已有索引文件（最后更新: [时间]）
     选择操作：
       1. 重建索引（删除旧索引，重新构建）
       2. 更新索引（增量更新，仅处理变更）
     ```

3. **开始构建**:
   ```
   开始构建索引...
     [1/5] 扫描文档...
     [2/5] 检测工具...
     [3/5] 转换文档...
     [4/5] 生成摘要...
     [5/5] 写入索引...
   ```

4. **输出结果**:
   ```yaml
   ✓ 索引构建完成
     - 知识库: [名称]
     - 文档数量: [数量]
     - 索引文件: [路径]
   ```

## 场景 2: 更新索引

### 决策树

```
用户请求更新索引
    │
    ├─ 关键词识别
    │   • "更新索引"、"同步索引"、"刷新索引"
    │   • "检查更新"
    │
    ├─ 检查现有索引
    │   ├─ 索引不存在
    │   │   └─ 提示: "索引不存在，是否建立新索引？"
    │   │       ├─ 是 → 执行场景1（建立索引）
    │   │       └─ 否 → 结束
    │   └─ 索引存在 → 继续
    │
    ├─ 选择检测方法
    │   ├─ 配置指定方法 → 使用配置方法
    │   ├─ 文档数量 < 100 → 使用 hash（精确）
    │   └─ 文档数量 ≥ 100 → 使用 mtime（快速）
    │
    ├─ 执行增量更新
    │   ├─ 读取现有索引
    │   ├─ 扫描当前文件
    │   ├─ 检测变更
    │   ├─ 处理变更
    │   └─ 更新索引文件
    │
    └─ 输出更新报告
```

### AI 行动指南

**检测到用户请求更新索引时**:

1. **输出确认信息**:
   ```
   ✓ 收到请求：更新「[知识库名]」的索引
   ```

2. **检查索引存在性**:
   - 如果索引不存在，输出:
     ```
     ⚠️ 索引文件不存在
     是否建立新索引？
     ```

3. **开始更新**:
   ```
   开始更新索引...
     [1/4] 读取现有索引...
     [2/4] 检测变更（使用 [mtime/hash] 方法）...
     [3/4] 处理变更...
       - 新增: [数量] 个
       - 修改: [数量] 个
       - 删除: [数量] 个
     [4/4] 更新索引文件...
   ```

4. **输出结果**:
   ```yaml
   ✓ 索引更新完成
     - 新增: [数量] 个文档
     - 修改: [数量] 个文档
     - 删除: [数量] 个文档
     - 当前总数: [数量] 个文档
   ```

## 场景 3: 智能检索

### 决策树

```
用户基于知识库提问或请求内容
    │
    ├─ 关键词识别
    │   • "在知识库中查找..."
    │   • "参考知识库，帮我写..."
    │   • "根据知识库，总结..."
    │   • "基于...写一份文档"
    │
    ├─ 识别目标知识库
    │   ├─ 明确指定 → 使用指定知识库
    │   ├─ 隐含指定（如"信息化"）→ 匹配知识库
    │   └─ 未指定 → 搜索所有知识库
    │
    ├─ 检查索引
    │   ├─ 索引不存在
    │   │   └─ 询问: "知识库未建立索引，是否先建立索引？"
    │   │       ├─ 是 → 执行场景1
    │   │       └─ 否 → 直接搜索文件（性能较差）
    │   └─ 索引存在 → 继续
    │
    ├─ 检查索引新鲜度
    │   ├─ 索引过期（> 7天）→ 警告用户
    │   └─ 索引新鲜 → 继续
    │
    ├─ 执行检索
    │   ├─ 读取索引
    │   ├─ 关键词匹配
    │   ├─ 语义匹配
    │   ├─ 排序结果
    │   └─ 读取相关文档
    │
    └─ 生成输出
        ├─ 回答问题
        └─ 创建新文档（如请求）
```

### AI 行动指南

**检测到用户基于知识库请求内容时**:

1. **识别意图**:
   - 提问 → 生成回答
   - 写作请求 → 生成新文档
   - 总结请求 → 生成总结

2. **输出检索信息**:
   ```
   ✓ 在「[知识库名]」中检索相关内容...
     [1/3] 读取索引...
     [2/3] 匹配相关文档...
       - 找到 [数量] 个相关文档
     [3/3] 读取文档内容...
   ```

3. **输出匹配结果**:
   ```
   相关文档:
     1. [文档名1] (相关度: 5/5)
     2. [文档名2] (相关度: 4/5)
     3. [文档名3] (相关度: 3/5)
   ```

4. **生成输出**:
   - 对于提问:
     ```
     基于知识库内容，回答如下：
     [答案内容]

     参考:
       - [文档名1]
       - [文档名2]
     ```

   - 对于写作请求:
     ```
     已基于知识库内容生成新文档
     参考:
       - [文档名1]
       - [文档名2]
     ```

## 场景 4: 查看索引状态

### 决策树

```
用户请求查看索引状态
    │
    ├─ 关键词识别
    │   • "查看索引状态"、"索引信息"
    │   • "检查哪些知识库需要更新"
    │   • "知识库状态"
    │
    ├─ 检查索引存在性
    │   ├─ 索引存在 → 读取索引
    │   └─ 索引不存在 → 提示用户
    │
    ├─ 对比当前状态
    │   ├─ 扫描当前文件
    │   └─ 对比索引中的文档列表
    │
    └─ 输出状态报告
```

### AI 行动指南

**检测到用户请求查看索引状态时**:

1. **输出检查信息**:
   ```
   ✓ 检查「[知识库名]」的索引状态...
   ```

2. **输出状态报告**:
   ```yaml
   知识库: [名称]
   索引状态:
     ✓ 索引文件存在
     ✓ 最后更新: [时间]
     ✓ 文档总数: [数量]
     ✓ 总大小: [大小]

   待更新:
     - 新增: [数量] 个文件未索引
     - 修改: [数量] 个文件已变更
     - 删除: [数量] 个文件已移除

   建议: [如有待更新，建议运行「更新索引」]
   ```

## 混合场景处理

### 场景: 用户未明确指定操作

**示例**: "我有一个「项目文档」文件夹，里面有各种文档"

**决策流程**:

```
用户提及知识库但未明确操作
    │
    ├─ 分析用户意图
    │   ├─ 可能是建立索引 → 询问确认
    │   ├─ 可能是查询内容 → 询问具体需求
    │   └─ 不明确 → 提供选项
    │
    └─ 提供选项
        "检测到您有知识库相关需求，我可以帮您：
          1. 建立知识库索引
          2. 在知识库中查找内容
          3. 查看索引状态
        请问您需要哪项操作？"
```

## 错误处理决策

### 错误场景 1: 路径不存在

```
检测到路径不存在
    │
    └─ 输出错误信息 + 询问
        "错误: 路径「[路径]」不存在
         请确认路径是否正确，或提供正确的路径。"
```

### 错误场景 2: 工具不可用

```
检测到无转换工具
    │
    ├─ 输出警告
    │   "⚠️ 未检测到文档转换工具
    │    仅索引 Markdown 文档"
    │
    └─ 提供安装建议
        "推荐安装工具以支持 PDF/Word:
          - doc2md skill
          - pandoc
          - PyMuPDF"
```

### 错误场景 3: 索引损坏

```
检测到索引损坏
    │
    ├─ 尝试修复
    │   ├─ 修复成功 → 继续
    │   └─ 修复失败 → 询问用户
    │
    └─ 询问用户
        "索引文件损坏，无法修复。
         是否重建索引？"
```

## 优先级决策

### 多个知识库时的优先级

```
检测到多个知识库
    │
    ├─ 用户明确指定 → 使用指定知识库
    ├─ 只有一个有索引 → 使用有索引的
    ├─ 多个有索引 → 询问用户
    └─ 都无索引 → 询问用户
```

### 工具选择优先级

```
检测到多个工具可用
    │
    ├─ 配置指定 → 使用配置工具
    └─ 自动选择
        1. doc2md skill (质量最高)
        2. pandoc (兼容性最好)
        3. PyMuPDF (速度最快)
        4. 直接读取 (Markdown)
```

## 决策辅助问题

当 AI Agent 不确定用户意图时，使用以下问题澄清:

### 澄清问题库

1. **操作类型不明确**:
   ```
   "请问您需要：
     1. 为知识库建立索引
     2. 更新现有索引
     3. 在知识库中查找内容
     4. 查看索引状态"
   ```

2. **知识库不明确**:
   ```
   "检测到多个知识库：
     - [知识库1]
     - [知识库2]
   请问您要操作哪个知识库？"
   ```

3. **索引已存在**:
   ```
   "索引已存在（最后更新: [时间]）
   请问您要：
     1. 重建索引（删除旧索引，重新构建）
     2. 更新索引（增量更新）"
   ```

4. **工具不可用**:
   ```
   "未检测到文档转换工具，仅能索引 Markdown 文档。
   是否继续？"
   ```
