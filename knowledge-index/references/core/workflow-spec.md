# 完整工作流程

本文档详细描述知识库索引的完整工作流程。

## 流程概览

```
首次构建 → 增量更新 → 智能检索 → 维护优化
```

## 1. 首次构建索引

### 触发条件

用户请求：
- "为「文件夹名」建立知识库索引"
- "帮我索引这个文件夹"
- "创建知识库"

### 执行流程

```
┌─────────────────────────────────────────┐
│ Step 1: 扫描文档                          │
├─────────────────────────────────────────┤
│ • 递归扫描知识库目录                      │
│ • 识别支持的文档格式                      │
│ • 排除配置中指定的文件/文件夹              │
│ • 统计文档数量和大小                      │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 2: 转换文档                          │
├─────────────────────────────────────────┤
│ • Markdown: 直接读取                     │
│ • PDF/Word: 使用工具转换                 │
│ • 检测可用工具（doc2md/pandoc/mineru）   │
│ • 工具不可用时降级处理                    │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 3: 生成摘要                          │
├─────────────────────────────────────────┤
│ • 读取文档内容                            │
│ • AI 提取核心观点和结构                   │
│ • 生成简洁摘要（100-300字符）             │
│ • 提取关键词和主题标签                    │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 4: 写入索引                          │
├─────────────────────────────────────────┤
│ • 构建 YAML 结构                         │
│ • 填充元数据和文档索引                    │
│ • 生成主题聚类（可选）                    │
│ • 写入 _index.yaml 文件                  │
└─────────────────────────────────────────┘
```

### 输出

```yaml
✓ 索引构建完成
  - 知识库: 信息化
  - 文档数量: 15
  - 总大小: 2.5 MB
  - 索引文件: 信息化/_index.yaml
```

```

           ↓
┌─────────────────────────────────────────┐
│ Step 2: 检测变更                          │
├─────────────────────────────────────────┤

│ 方法2: hash（准确）                      │

└─────────────────────────────────────────┘
           ↓

├─────────────────────────────────────────┤
│ • 新增文档: 转换 → 摘要 → 添加到索引     │

└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 4: 更新索引文件                      │
├─────────────────────────────────────────┤

│ • 更新 last_updated 时间戳               │
│ • 写入 _index.yaml                      │
└─────────────────────────────────────────┘
```

### 变更检测算法

#### mtime 方法（默认）

```python
# 伪代码
for file in current_files:
    if file not in index:
        → 新增
    elif file.mtime > index[file].modified:
        → 修改

for file in index:
    if file not in current_files:
        → 删除
```

#### hash 方法（精确）

```python
# 伪代码
for file in current_files:
    current_hash = calculate_hash(file)
    if file not in index:
        → 新增
    elif current_hash != index[file].hash:
        → 修改

for file in index:
    if file not in current_files:
        → 删除
```

### 输出

```yaml
✓ 索引更新完成
  - 新增: 3 个文档
  - 修改: 2 个文档
  - 删除: 1 个文档
  - 当前总数: 17 个文档
```

## 3. 智能检索

### 触发条件

用户请求：
- "在知识库中查找关于 XXX 的内容"
- "帮我写一份关于 YYY 的文档，参考知识库"
- "根据知识库，总结 ZZZ"

### 执行流程

```
┌─────────────────────────────────────────┐
│ Step 1: 读取索引                          │
├─────────────────────────────────────────┤
│ • 加载 _index.yaml                      │
│ • 解析所有文档的摘要和关键词              │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 2: 匹配相关文档                      │
├─────────────────────────────────────────┤
│ 方法1: 关键词匹配                        │
│   • 用户问题提取关键词                   │
│   • 与文档 keywords/topics 匹配         │
│                                         │
│ 方法2: 语义相似度（AI）                  │
│   • 理解用户意图                         │
│   • AI 评估文档相关性                    │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 3: 读取文档全文                      │
├─────────────────────────────────────────┤
│ • 读取匹配度最高的 N 个文档              │
│ • 转换为可读格式                         │
│ • 提取相关章节                           │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Step 4: 生成回答                          │
├─────────────────────────────────────────┤
│ • 基于文档内容生成回答                   │
│ • 引用来源文档                           │
│ • 如需创建新文档，整合多个来源           │
└─────────────────────────────────────────┘
```

### 匹配算法

#### 关键词匹配

```python
# 伪代码
query_keywords = extract_keywords(user_query)
matched_docs = []

for doc in index.documents:
    score = 0
    for keyword in query_keywords:
        if keyword in doc.keywords:
            score += 2
        if keyword in doc.topics:
            score += 1
    if score > 0:
        matched_docs.append((doc, score))

return sorted(matched_docs, key=lambda x: x[1], reverse=True)
```

#### 语义匹配（AI）

```
AI 分析用户问题和每个文档摘要的相关性
评分标准：
  - 高度相关: 5分
  - 部分相关: 3分
  - 可能相关: 1分
  - 不相关: 0分
返回评分最高的前 N 个文档
```

### 输出

```
找到 3 个相关文档：
1. GitLab 完整指南.md (相关度: 5)
2. CI/CD 最佳实践.pdf (相关度: 4)
3. DevOps 工具链.docx (相关度: 3)

[基于文档内容生成回答或新文档]
```

## 异常处理

### 文档转换失败

```
检测到问题 → 跳过该文档 → 记录错误 → 继续处理其他文档
```

### 工具不可用

```
首选工具不可用 → 尝试备用工具 → 全部不可用时仅索引 Markdown
```

### 索引文件损坏

```
读取失败 → 提示用户 → 建议重建索引
```

## 性能优化

### 大规模知识库（>1000 文档）

- 分批处理：每批 50-100 个文档
- 并行转换：多线程处理
- 进度显示：每批完成后报告进度
- 索引压缩：删除冗余字段

详见 `references/advanced/performance-tuning.md`
