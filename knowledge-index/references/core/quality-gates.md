# 质量门禁

本文档定义知识库索引构建和更新的质量检查点。

## 三层验证机制

```
Layer 1: 结构验证 → Layer 2: 语义验证 → Layer 3: 完整性验证
```

## 1. 构建前检查（Pre-build）

### 1.1 环境检查

- [ ] **知识库路径有效**
  - 路径存在且可访问
  - 路径不是根目录或系统目录

- [ ] **至少有一个支持的文档**
  - 存在 .md/.pdf/.docx/.doc 文件
  - 文件数量 > 0

- [ ] **工具可用性检测**
  - 至少有一个文档转换工具可用
  - 如无工具，仅索引 Markdown 文档并提示

**失败处理**: 输出具体错误信息，停止构建

### 1.2 配置验证

- [ ] **配置文件格式正确**（如存在 `_index_config.yaml`）
  - YAML 语法正确
  - 配置项合法

- [ ] **排除规则有效**
  - 排除的文件/文件夹存在（避免排除所有文件）
  - 正则表达式语法正确

**失败处理**: 使用默认配置，警告用户

## 2. 构建中检查（In-build）

### 2.1 文档转换检查

- [ ] **文档成功转换**
  - 转换后内容非空
  - 转换后内容长度 > 100 字符
  - 无乱码或编码错误

- [ ] **转换失败处理**
  - 记录失败的文件路径
  - 跳过该文件，继续处理
  - 在最终报告中列出失败文件

**处理策略**: 降级处理，不中断构建流程

### 2.2 AI 摘要生成检查

- [ ] **摘要长度合理**
  - 最小: 50 字符
  - 最大: 500 字符（或配置值）
  - 推荐: 100-300 字符

- [ ] **摘要质量**
  - 非空内容
  - 包含文档的核心信息
  - 结构化（分点、编号）更佳

- [ ] **关键词提取**
  - 数量: 5-10 个
  - 非通用词（如"文档"、"说明"）
  - 具有检索价值

**失败处理**: 重新生成一次，仍失败则使用文档前 200 字符作为摘要

### 2.3 内存和性能监控

- [ ] **内存使用正常**
  - 单个文档处理内存 < 100MB
  - 总内存使用 < 可用内存的 80%

- [ ] **处理时间合理**
  - 单个文档处理时间 < 30 秒
  - 超时文档跳过并记录

**失败处理**: 分批处理，或启用性能优化模式

## 3. 构建后检查（Post-build）

### 3.1 索引文件验证

- [ ] **YAML 语法正确**
  - 文件可被 YAML 解析器读取
  - 无语法错误

- [ ] **必填字段完整**
  - 元数据字段全部存在
  - 每个文档包含必填字段

- [ ] **字段类型正确**
  - 时间格式符合 ISO 8601
  - 数值字段为数值类型
  - 数组字段为 YAML 数组

**验证脚本示例**:

```python
import yaml
from datetime import datetime

def validate_index(index_path):
    with open(index_path, encoding='utf-8') as f:
        index = yaml.safe_load(f)

    # 检查元数据
    assert 'version' in index
    assert 'knowledge_base' in index
    kb = index['knowledge_base']
    assert 'name' in kb
    assert 'total_documents' in kb

    # 检查文档
    for doc in index['documents']:
        assert 'path' in doc
        assert 'filename' in doc
        assert 'type' in doc
        assert 'summary' in doc
        assert len(doc['summary']) >= 50

        # 验证时间格式
        datetime.fromisoformat(doc['modified'].replace('Z', '+00:00'))

    print("✓ 索引验证通过")
```

### 3.2 统计一致性

- [ ] **文档数量一致**
  - `total_documents` = len(`documents`)

- [ ] **总大小一致**
  - `total_size_mb` ≈ sum(所有文档 size) / 1024 / 1024

- [ ] **时间戳更新**
  - `last_updated` >= `created`
  - `last_updated` 接近当前时间

**失败处理**: 自动修正统计信息，重新写入索引

### 3.3 语义完整性

- [ ] **所有文档已索引**
  - 索引中的文档数量 ≥ 知识库中支持的文档数量 × 0.9
  - 允许少量转换失败

- [ ] **主题聚类合理**（如启用）
  - 每个聚类包含至少 2 个文档
  - 每个文档属于至少 1 个聚类

- [ ] **路径有效性**
  - 索引中的路径使用正斜杠
  - 路径相对于知识库根目录

**失败处理**: 警告用户，但不阻止使用

## 4. 增量更新检查（Update）

### 4.1 变更检测验证

- [ ] **检测方法有效**
  - mtime: 文件系统支持修改时间
  - hash: 计算准确

- [ ] **检测结果合理**
  - 新增/修改/删除的文档数量在预期范围
  - 无异常的大量变更（如 90% 文档变更）

**失败处理**: 提示用户确认，或切换检测方法

### 4.2 合并验证

- [ ] **合并后文档数量正确**
  - 新文档数量 = 旧文档数量 + 新增 - 删除

- [ ] **文档顺序保持**
  - 现有文档保持原有顺序
  - 新增文档追加到末尾

**失败处理**: 回滚到旧索引，报告错误

## 5. 智能检索检查（Search）

### 5.1 索引加载检查

- [ ] **索引文件存在**
  - `_index.yaml` 存在于知识库根目录

- [ ] **索引未过期**
  - `last_updated` 距今不超过配置的过期时间（如 7 天）

**失败处理**: 提示用户更新索引

### 5.2 匹配质量检查

- [ ] **匹配结果相关**
  - 前 3 个结果的相关性评分 > 阈值
  - 无完全不相关的结果

- [ ] **召回率合理**
  - 相关文档未被遗漏

**失败处理**: 调整匹配算法，或提示用户优化查询

  - 关键词: 5-8 个/文档
