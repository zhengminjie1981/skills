# 检查清单

本文档提供知识库索引操作的检查清单，确保每个步骤正确执行。

## 首次构建索引检查清单

### 构建前准备

- [ ] **确认知识库路径**
  - 路径存在且可访问
  - 路径指向正确的知识库目录
  - 有足够的磁盘空间存储索引文件

- [ ] **扫描文档**
  - 识别所有支持的文档格式（.md, .pdf, .docx, .doc）
  - 排除不需要索引的文件（临时文件、草稿等）
  - 统计文档数量和总大小

- [ ] **检测转换工具**
  - 检查 doc2md skill 是否可用
  - 检查系统是否安装 pandoc
  - 检查 Python 包 PyMuPDF/mineru 是否安装
  - 至少有一个工具可用（或仅索引 Markdown）

- [ ] **加载配置**（如存在）
  - 读取 `_index_config.yaml`
  - 验证配置格式
  - 应用配置到构建流程

### 构建执行

- [ ] **文档转换**
  - Markdown 文档直接读取
  - PDF/Word 文档使用工具转换
  - 记录转换失败的文档
  - 验证转换后内容非空

- [ ] **AI 摘要生成**
  - 提取文档核心内容
  - 生成简洁摘要（100-300 字符）
  - 提取关键词（5-10 个）
  - 提取主题标签（3-5 个）

- [ ] **索引构建**
  - 填充元数据（名称、路径、创建时间等）
  - 构建文档列表
  - 生成主题聚类（可选）
  - 计算统计信息

- [ ] **写入索引文件**
  - YAML 格式化
  - UTF-8 编码
  - 写入 `_index.yaml`

### 构建后验证

- [ ] **YAML 语法验证**
  - 文件可被 YAML 解析器读取
  - 无语法错误

- [ ] **字段完整性验证**
  - 所有必填字段存在
  - 字段类型正确
  - 时间格式符合 ISO 8601

- [ ] **统计一致性验证**
  - `total_documents` = len(`documents`)
  - `total_size_mb` 计算正确
  - `last_updated` 已更新

- [ ] **功能测试**
  - 索引文件可被读取
  - 智能检索功能正常
  - 索引状态查询正常

### 输出报告

```yaml
构建报告:
  知识库: [名称]
  路径: [路径]

  文档统计:
    - 总数: [数量]
    - Markdown: [数量]
    - PDF: [数量]
    - Word: [数量]
    - 转换失败: [数量]

  索引文件:
    - 位置: [路径]/_index.yaml
    - 大小: [KB]
    - 创建时间: [时间]

  质量评分:
    - 转换成功率: [%]
    - 摘要平均长度: [字符]
    - 关键词平均数量: [个]

  建议:
    - [如有问题，列出建议]
```

## 增量更新索引检查清单

### 更新前准备

- [ ] **读取现有索引**
  - `_index.yaml` 文件存在
  - 文件可被正常解析
  - 索引未损坏

- [ ] **选择检测方法**
  - mtime 方法（快速，适合大多数场景）
  - hash 方法（精确，适合需要精确检测的场景）
  - 混合方法（mtime 初筛 + hash 精确验证）

### 变更检测

- [ ] **扫描当前文件**
  - 递归扫描知识库目录
  - 识别所有支持的文档
  - 应用排除规则

- [ ] **对比变更**
  - 识别新增文档
  - 识别修改文档
  - 识别删除文档
  - 记录变更列表

- [ ] **验证检测结果**
  - 变更数量在合理范围
  - 无异常的大量变更（如 90% 文档变更）

### 增量处理

- [ ] **处理新增文档**
  - 转换文档
  - 生成摘要
  - 添加到索引末尾

- [ ] **处理修改文档**
  - 重新转换
  - 重新生成摘要
  - 更新索引中的对应条目

- [ ] **处理删除文档**
  - 从索引中移除
  - 更新统计信息

- [ ] **更新元数据**
  - 更新 `last_updated` 时间戳
  - 重新计算 `total_documents`
  - 重新计算 `total_size_mb`
  - 重新生成主题聚类（可选）

### 更新后验证

- [ ] **文档数量验证**
  - 新文档数 = 旧文档数 + 新增 - 删除

- [ ] **字段更新验证**
  - `last_updated` 已更新
  - 统计信息已更新

- [ ] **YAML 语法验证**
  - 更新后的索引文件语法正确

### 输出报告

```yaml
更新报告:
  知识库: [名称]

  变更统计:
    - 新增: [数量] 个文档
    - 修改: [数量] 个文档
    - 删除: [数量] 个文档
    - 当前总数: [数量] 个文档

  处理结果:
    - 成功: [数量]
    - 失败: [数量]
    - 失败原因: [列表]

  索引文件:
    - 最后更新: [时间]
    - 文件大小: [KB]
```

## 智能检索检查清单

### 检索前准备

- [ ] **索引文件存在**
  - `_index.yaml` 存在于知识库根目录
  - 文件可被正常读取

- [ ] **索引未过期**
  - `last_updated` 距今不超过配置的过期时间
  - 如过期，提示用户更新索引

- [ ] **解析用户查询**
  - 提取关键词
  - 理解用户意图
  - 确定查询范围

### 检索执行

- [ ] **关键词匹配**
  - 在 `keywords` 字段中搜索
  - 在 `topics` 字段中搜索
  - 在 `summary` 字段中搜索

- [ ] **语义匹配**（AI）
  - 理解用户问题和文档摘要的语义相关性
  - 为每个文档评分

- [ ] **排序结果**
  - 按相关度评分排序
  - 返回前 N 个结果

- [ ] **读取文档全文**
  - 读取匹配度最高的文档
  - 转换为可读格式
  - 提取相关章节

### 输出生成

- [ ] **生成回答**
  - 基于文档内容生成回答
  - 引用来源文档
  - 标注文档路径

- [ ] **创建新文档**（如需要）
  - 整合多个来源
  - 生成新文档
  - 引用所有参考文档

### 输出报告

```yaml
检索报告:
  查询: [用户查询]

  匹配结果:
    - 文档1: [路径] (相关度: [评分])
    - 文档2: [路径] (相关度: [评分])
    - 文档3: [路径] (相关度: [评分])

  使用文档:
    - [路径1]
    - [路径2]

  输出:
    - 类型: [回答/新文档]
    - 引用来源: [数量] 个文档
```

## 索引状态检查清单

### 状态查询

- [ ] **读取索引**
  - 加载 `_index.yaml`
  - 解析元数据和文档列表

- [ ] **扫描当前文件**
  - 递归扫描知识库
  - 统计当前文档

- [ ] **对比差异**
  - 识别新增文件
  - 识别修改文件
  - 识别删除文件

### 输出报告

- [ ] **基本信息**
  - 知识库名称
  - 索引文件位置
  - 最后更新时间

- [ ] **文档统计**
  - 文档总数
  - 按类型统计
  - 总大小

- [ ] **待更新列表**
  - 新增文件列表
  - 修改文件列表
  - 删除文件列表

- [ ] **维护建议**
  - 是否需要更新
  - 推荐的更新频率
  - 其他优化建议
