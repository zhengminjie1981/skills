## 核心设计决策

### 1. 知识库定义

**一个知识库 = 一个文件夹 + 一个索引文件**

```
知识库/
├── _index.yaml          # 知识库索引（覆盖所有子文件夹）
├── 文档1.md
└── 子文件夹/
    └── 文档2.md         # 也属于该知识库
```

**关键原则**：
- ✅ 一个索引文件覆盖该文件夹及其所有子文件夹
- ❌ 不在子文件夹创建独立索引
- ✅ 索引文件在知识库内部（便于版本控制）

---

### 2. 全局注册表

**位置**: `~/.knowledge-index/registry.yaml`

**作用**：集中管理所有知识库的元信息

**核心功能**：
- 快速发现所有知识库
- 跨知识库检索
- 层级冲突检测
- 知识库健康检查

**数据结构**：
```yaml
version: "1.0"
last_updated: "2026-03-04T10:00:00Z"
knowledge_bases:
  - name: "信息化知识库"
    path: "E:/知识库/信息化"
    index_path: "E:/知识库/信息化/_index.yaml"
    created: "2026-02-15T10:00:00Z"
    last_updated: "2026-03-04T09:30:00Z"
    document_count: 15
    total_size_mb: 2.5
    status: "active"
    hierarchy:
      parent: null
      depth: 0
      children: []
```

---

### 3. 层级提升策略

**核心原则**: 始终在最高级别维护索引

**策略**：检测到父子索引冲突时，自动删除下级索引，保留最高级别索引

---

## 8 种场景处理逻辑

| # | 当前 | 父 | 子 | 操作 | 说明 |
|---|------|----|----|------|------|
| 1 | 无 | 无 | - | ✅ 创建索引 | 标准创建 |
| 2 | 有 | 无 | - | ✅ 更新索引 | 正常更新 |
| 3 | 无 | 有 | - | ⚠️ 建议更新父索引 | 避免重复 |
| 4 | 有 | 有 | - | 🔄 提升至父 | 删除当前 |
| 5 | 无 | 无 | 有 | 🔄 提升至当前 | 删除子索引 |
| 6 | 有 | 无 | 有 | 🔄 合并子索引 | 删除子索引 |
| 7 | 无 | 有 | 有 | 🔄 提升至父 | 删除当前和子 |
| 8 | 有 | 有 | 有 | 🔄 提升至父 | 删除所有下级 |

## 核心算法

### 决策树

```python
def build_index(kb_path):
    parent_index = find_parent_index(kb_path)
    current_index = has_index(kb_path)
    child_indexes = find_child_indexes(kb_path)

    if parent_index:
        if current_index:
            # 场景 4, 8: 层级提升至父
            return promote_to_parent(kb_path, parent_index)
        else:
            # 场景 3, 7: 建议更新父索引
            return suggest_update_parent(kb_path, parent_index)
    else:
        if child_indexes:
            # 场景 5, 6: 层级提升至当前
            return promote_to_current(kb_path, child_indexes)
        else:
            # 场景 1, 2: 正常创建或更新
            if current_index:
                return update_index(kb_path)
            else:
                return create_index(kb_path)
```

### 层级提升流程

```
检测到层级冲突
    │
    ├─ [1/4] 备份索引
    │   - 备份当前索引
    │   - 备份所有子索引
    │
    ├─ [2/4] 删除下级索引
    │   - 删除当前索引
    │   - 删除所有子索引
    │   - 从注册表移除
    │
    ├─ [3/4] 更新父索引
    │   - 扫描所有文档
    │   - 重新生成索引
    │   - 更新注册表
    │
    └─ [4/4] 验证索引
        - 验证文件存在
        - 验证字段完整
        - 验证注册表同步
```

## 风险与缓解

### 风险 1: 误删索引

**场景**: 用户不知情的情况下删除了子索引

**缓解**:
- ✅ 删除前备份
- ✅ 明确的警告提示
- ✅ 可配置为 manual 模式

### 风险 2: 注册表损坏

**场景**: 注册表文件损坏或丢失

**缓解**:
- ✅ 自动备份机制
- ✅ 重建注册表功能
- ✅ 索引文件仍可独立使用

### 风险 3: 路径移动

**场景**: 知识库移动后路径不匹配

**缓解**:
- ✅ 健康检查检测
- ✅ 自动更新注册表路径
- ✅ 提示用户重新索引
