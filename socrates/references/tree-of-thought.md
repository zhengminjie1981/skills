# 思维树探索方法

## 适用场景

- 需要在多个选项间做决策
- 问题有多种可能解释
- 不确定哪条路径最优

## 基本结构

```
问题/决策点
├── 路径 A：假设 XXX
│   ├── 如果成立 → 结果 A1
│   └── 如果不成立 → 结果 A2
├── 路径 B：假设 YYY
│   ├── 如果成立 → 结果 B1
│   └── 如果不成立 → 结果 B2
└── 🔑 关键验证点：如何判断 A/B 哪个成立？
```

## 实施步骤

1. **明确核心问题** - 一句话定义
2. **列出主要路径** - 2-4 个选择/假设分支
3. **推演每条路径** - 优点、代价、适用条件、风险
4. **识别关键假设** - 决定路径选择的关键变量
5. **设定验证方法** - 如何验证假设

## 输出模板

```markdown
## 问题：[一句话]

### 路径对比

| 维度 | 路径 A | 路径 B | 路径 C |
|------|--------|--------|--------|
| 核心假设 | | | |
| 主要优点 | | | |
| 主要风险 | | | |
| 适用条件 | | | |

### 关键假设

| 假设 | 如果成立 | 如果不成立 | 如何验证 |
|------|----------|------------|----------|
| 1 | 路径 A | 路径 B | [方法] |

### 建议

- 如果 [条件X]，选择路径 A
- 如果 [条件Y]，选择路径 B

### 下一步验证

- [ ] [验证行动1]
- [ ] [验证行动2]
```

## 实例：职业选择

```markdown
## 问题：留现公司还是跳槽？

### 路径对比

| 维度 | 留现公司 | 跳槽创业公司 | 跳槽大厂 |
|------|----------|--------------|----------|
| 核心假设 | 公司有成长空间 | 公司靠谱、股权有价值 | 能拿到核心组 offer |
| 主要优点 | 稳定、人脉积累 | 成长快、股权 | 薪资高、平台大 |
| 主要风险 | 错失外部机会 | 失败率高 | 晋升慢、螺丝钉 |

### 关键假设

| 假设 | 成立→ | 不成立→ | 验证方法 |
|------|-------|---------|----------|
| 现公司 3 年内 IPO | 留 | 跳 | 了解公司财务状况 |
| 能拿到大厂核心组 | 考虑大厂 | 排除大厂 | 面试看反馈 |

### 建议

- 现公司 IPO 可能性 > 50% 且个人发展顺畅 → 留
- 大厂能拿到核心组 offer → 优先大厂
- 追求高风险高回报且找到靠谱创业公司 → 创业公司
```
