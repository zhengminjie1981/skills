# 结构化思维框架

## 5 Whys - 根因分析

连续追问"为什么"，穿透表象找到根本原因。

```
问题：网站加载慢
为什么？→ 数据库查询慢
为什么？→ 缺少索引
为什么？→ 开发时没考虑性能
为什么？→ 没有 DBA 参与 schema 设计
     ↓
根因：开发流程缺少数据库设计评审
```

**注意**: 每一层"为什么"都要基于事实，根因应该是**可行动**的。

---

## MECE - 问题拆解

**M**utually **E**xclusive, **C**ollectively **E**xhaustive（相互独立，完全穷尽）

```
收入下降
├── 用户数下降
│   ├── 新用户减少
│   └── 老用户流失
└── 客单价下降
    ├── 购买频次降低
    └── 单次购买金额降低
```

**检验方法**:
- 独立性：A 和 B 有交集吗？
- 穷尽性：除了这些，还有遗漏吗？

---

## First Principles - 第一性原理

拆解到最基本的事实，从零开始重建。

```
类比思维：别人怎么做，我怎么做
     ↓
第一性原理：
  - 基本构成是什么？
  - 哪些是物理/逻辑约束，哪些只是惯例？
  - 如果从零开始设计，会怎么做？
```

---

## Inversion - 逆向思维

与其思考"如何成功"，不如思考"如何避免失败"。

```
正向：如何让产品成功？
逆向：什么会导致产品失败？
      - 解决不存在的问题
      - 目标用户不清晰
      - 核心价值不突出
      → 确保不犯这些错误
```

---

## Second-Order Thinking - 二阶思维

不只考虑直接后果，还要考虑后果的后果。

```
决策：降价促销
一阶：销量↑ 收入↑
二阶：用户认为原价虚高 → 后续不打折没人买
三阶：品牌价值受损
```

---

## 框架选择

| 场景 | 框架 |
|------|------|
| 找根因 | 5 Whys |
| 全面拆解 | MECE |
| 突破常规 | First Principles |
| 识别风险 | Inversion |
| 评估影响 | Second-Order |

**组合使用**: MECE 拆解 → 5 Whys 找根因 → First Principles 突破 → Inversion 识别风险
