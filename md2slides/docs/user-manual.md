## 1. 快速入门

### md2slides 能做什么

md2slides 是一个 Claude Code skill，帮助你将**原始材料**（报告、文章、数据、笔记）转换为可在浏览器中演示的 HTML 幻灯片，并支持导出 PDF。

**核心能力一览**：

| 能力 | 说明 |
|------|------|
| 智能策划 | AI 分析材料，自动规划页面结构和内容分配 |
| 版式规划 | AI 从 27 种布局模板中为每页选定最佳版式 |
| 数据图表 | 自动识别数据并生成柱状图、折线图、饼图等 |
| 原生表格 | 文字型数据直接渲染为带样式的 HTML 表格 |
| 主题风格 | 5 套预设主题，支持 16:9 / 4:3 两种比例 |
| 键盘翻页 | 方向键和空格键控制翻页，右下角显示页码 |
| 精细调整 | 通过对话调整任意元素的样式，或切换版式 |
| PDF 导出 | 自动检测 Edge/Chrome，每张幻灯片对应一页 PDF |

### 典型工作流

```
你提供材料
    ↓
阶段一：AI 策划 MD 内容（你可以修改）
    ↓
阶段二：AI 版式规划（AI 为每页选定最佳布局）
    ↓
阶段三：运行脚本生成 HTML（浏览器自动打开）
    ↓
按需调整：样式 / 内容 / 版式
    ↓
导出 PDF（可选）
```

### 如何触发 md2slides

直接用自然语言告诉 Claude 你要做什么：

```
"帮我把这份报告做成演示文稿"
"把 report.md 转成幻灯片"
"这些数据需要做一个 PPT 展示"
"帮我设计版式"
"第3页换成双栏布局"
```

## 2. 三阶段工作流详解

### 阶段一：内容策划（AI 对话）

AI 分析你提供的材料，规划演示结构，生成 MD 文件：

- 识别封面、目录、内容、数据、结语页面
- 数值型数据 → 提取为 CSV + `<!-- chart -->` 块
- 文字型表格 → 直接写 Markdown `|` 表格
- 展示策划摘要（页数 + 各页标题）等待你确认

### 阶段二：版式规划（AI 对话）

AI 读取已确认的 MD，逐页分析内容密度与元素构成，从 27 种版式模板中选定最佳布局，写入 `slide-tree.json`：

```json
{
  "index": 3,
  "type": "content",
  "layout": {
    "template": "image-right",
    "imageWidth": "38%",
    "rationale": "6条要点为主体，图片辅助视觉"
  }
}
```

AI 会告知每页的版式选择结果，你可以要求调整。

### 阶段三：HTML 生成（脚本）

确认版式后，AI 运行脚本：

```bash
python scripts/convert.py --input demo.md --output demo.html --tree slide-tree.json --serve
```

`--serve` 参数让生成完成后立即用系统浏览器打开 HTML 文件（file:// 协议，无需 HTTP 服务）。

## 3. 场景 A：从原始材料生成演示文稿

**适用情况**：你有一份文章、报告或笔记，想直接做成幻灯片。

### 步骤 1：提供材料

```
你：帮我把下面这份产品季报做成演示文稿：
[粘贴材料内容]
```

或指定文件路径：

```
你：帮我把 D:/work/Q4-report.md 做成演示文稿
```

### 步骤 2：确认内容策划

AI 展示策划结果：

```
已完成内容策划，共 9 页：

页码  类型    标题
1     封面    2024 Q4 产品季报
2     目录    目录
3     内容    季度概览
4     内容    用户增长
5     数据    月活跃用户趋势（含折线图）
6     内容    核心功能迭代
7     数据    收入结构（含柱状图）
8     内容    下季度计划
9     结语    谢谢 & 联系方式

数据文件：
- data/Q4-report-s5-users.csv（已创建）
- data/Q4-report-s7-revenue.csv（已创建）

请确认内容，或告知需要调整的地方。
```

你可以：
- 说"好的，继续" → 进入版式规划
- 提出修改意见：
  ```
  "第6页内容太少，拆成两页"
  "把目录页去掉"
  "第5页改用柱状图"
  "在第8页前加一页风险分析"
  ```

### 步骤 3：版式规划

AI 为每页选定版式，例如：

```
版式规划完成：
第1页  封面    → text-hero（无图片封面，突出标题）
第2页  目录    → text-default（标题+列表）
第3页  内容    → text-two-column（6条要点均分两列）
第4页  内容    → image-right（图片辅助，文字为主）
第5页  数据    → chart-full（折线图全屏）
第6页  内容    → text-default（3条要点）
第7页  数据    → chart-right（柱状图+左侧说明）
第8页  内容    → text-default
第9页  结语    → text-hero（突出感谢语）

是否确认，或需要调整某页版式？
```

### 步骤 4：选择主题（如未在材料中指定）

```
根据季报的商务汇报风格，建议使用 professional-dark（深色背景，蓝白文字）。
是否采用，或有其他偏好？
```

> **快捷方式**：说"用默认配置"可跳过宽高比和主题询问，直接使用 16:9 + professional-dark。

### 步骤 5：生成并查看

```
HTML 已生成：Q4-report.html（9 页）
正在用浏览器打开...

生成文件：
  Q4-report.html          （演示文稿）
  Q4-report-tree.json     （版式+元素树）
  data/Q4-report-s5-users.csv
  data/Q4-report-s7-revenue.csv
```

## 4. 场景 B：从已有 MD 文件生成演示文稿

**适用情况**：你已经有一个 Markdown 文件，想将它转换为 HTML 幻灯片。

### 方式一：AI 辅助转换（推荐）

```
你："把 slides-draft.md 转成演示文稿，帮我优化一下分页"
```

AI 读取 MD，重新策划内容，走场景 A 的标准三阶段流程。

### 方式二：直接转换（保留原有分页）

```
你："把 slides-draft.md 直接转成 HTML，不要改内容"
```

AI 按 MD 中已有的 `---` 分页符直接进入版式规划，只询问主题。

### MD 文件格式

```markdown
---
title: 演示文稿标题
ratio: 16:9
theme: professional-dark
---

<!-- Slide 1: 封面 -->
# 主标题
## 副标题

---

<!-- Slide 2: 内容页 -->
# 这一页的主题
- 要点一
- 要点二

---

<!-- Slide 3: 数据页 -->
# 用户增长趋势

<!-- chart
type: line
dataSource: data/demo-s3-users.csv
xField: 月份
yField: 用户数(万)
position: right
width: 58%
-->

- 说明文字一
- 说明文字二
```

**分栏语法（:::col）**：

多组并列内容可用 `:::col` 标记显式分列，无需在 slide-tree.json 中配置：

```markdown
:::col
**痛点 A**
- 信息分散，查找困难
- 缺乏统一入口
:::col
**痛点 B**
- 协作效率低
- 版本混乱
:::
```

两组内容自动排为两列，三组自动排为三列。

## 5. 场景 C：含数据的演示文稿

**适用情况**：材料中包含数据，需要在演示文稿中以图表或表格形式展示。

### 5.1 AI 自动识别数据

AI 根据数据特征自动决定处理方式：

| 数据特征 | 处理方式 |
|---------|---------|
| 数值型，3行以上，适合可视化 | 提取为 CSV + chart 块 |
| 多列结构化，需要原文对照 | 直接写 Markdown `\|` 表格 |
| 文字描述为主 / 数据点 < 3 | 保留为正文 |

### 5.2 图表数据（CSV + Chart.js）

AI 自动提取并生成 CSV，你不需要手动操作。如果提取的数据有误：

```
你："第5页的数据有问题，正确数据是：Q1=100, Q2=130, Q3=158, Q4=200"
```

或直接编辑 `data/` 目录下的 CSV 文件（UTF-8 编码，第一行为表头）。

**指定图表类型**：

```
"第5页的图改成饼图"
"用折线图展示用户趋势，柱状图展示收入对比"
"第7页不要图表，改成精确数值表格"
```

### 5.3 原生 MD 表格

文字型/混合型数据直接写标准 Markdown 表格，不需要 CSV，渲染为带样式的 HTML 表格：

```markdown
| 功能 | 基础版 | 专业版 | 企业版 |
|------|--------|--------|--------|
| 用户数 | ≤5人 | ≤50人 | 无限制 |
| 存储空间 | 5GB | 100GB | 1TB |
| 数据导出 | 不支持 | 支持 | 支持 |
```

版式规划时告诉 AI：

```
"这页的表格是功能对比，用 table-full 版式"
```

### 5.4 指定图表位置

```
"图在右边，文字在左"   → chart-right 版式
"图在左边，文字在右"   → chart-left 版式
"只显示图表"           → chart-full 版式
"图在下方，文字在上"   → chart-bottom 版式
```

## 6. 场景 D：内容变更

**适用情况**：HTML 已生成，需要修改演示文稿的**内容**（文字、列表、页面结构、图表类型）。

> **内容变更 vs 样式调整 vs 版式切换**
>
> | 修改类型 | 走哪个场景 |
> |---------|------------|
> | 文字错误、要点措辞、标题 | 内容变更（本场景） |
> | 增加/删除/移动页面 | 内容变更（本场景） |
> | 图表类型、数据源字段 | 内容变更（本场景） |
> | 字体大小、颜色、间距 | 样式调整（场景 E） |
> | 图表位置、宽高百分比 | 样式调整（场景 E） |
> | 换版式/换布局模板 | 版式切换（场景 F） |

### 6.1 修改文字内容

```
"第3页第二条要点改成'月活用户超过 30 万，同比增长 45%'"
"第5页标题改为'核心功能迭代'"
```

AI 定位 MD 文件中对应页面，修改文字，重新生成 HTML（保留已有样式）。

### 6.2 增加 / 删除列表条目

```
"第3页再加一条要点：'自然流量占比提升至 38%'"
"把第5页第三条删掉"
```

### 6.3 增加 / 删除 / 移动页面

```
"在第5页后面加一页'风险分析'，包含三条要点：..."
"删除第2页（目录页）"
"把第6页和第7页交换位置"
```

> **注意**：增/删页面会导致其后所有页面的页码改变，这些页的手工样式将被重置。
> AI 在执行前会告知影响范围，确认后再继续。

### 6.4 修改图表类型或数据

```
"第4页的柱状图改成折线图"
"第7页的图表换成展示精确数值的表格"
```

### 6.5 样式保留说明

| 情况 | 样式结果 |
|------|----------|
| 修改页面内容，页码不变 | 该页手工样式自动保留 |
| 增加新页（后续页码后移） | 后移页的手工样式丢失（AI 提前告知） |
| 删除页面（后续页码前移） | 后移页的手工样式丢失（AI 提前告知） |
| 修改内容但元素 ID 变化 | 该元素样式重置为主题默认值 |

## 7. 场景 E：样式调整

**适用情况**：HTML 已生成，需要修改某些页面或元素的外观（不改内容）。

用自然语言描述你要改什么，AI 自动定位并修改（**你不需要知道元素 ID**）。

### 7.1 文字样式

```
"第1页的标题字再大一点"
"第3页的标题改成黄色"
"第4页的列表字体加粗"
"第2页的列表文字颜色浅一点"
```

支持中文颜色名（白色、黑色、红色、蓝色、黄色、橙色、紫色、灰色……）和 CSS 色值（`#FF6B6B`、`rgba(255,255,255,0.8)`）。

**调整幅度参考**：

| 你说的 | 对应调整量 |
|-------|-----------|
| 大一点 / 小一点 | ±0.3~0.5em |
| 大很多 / 小很多 | ±0.8~1.2em |
| 加粗 | font-weight: bold |
| 细一点 | font-weight: 300 |
| 间距大一点 | margin/padding +0.3~0.5em |

### 7.2 布局调整

```
"第2页的列表行间距大一点"
"封面的文字居中对齐"
"第1页背景改成深蓝色"
```

### 7.3 图表样式调整

```
"第4页的图表移到左边"
"第5页的图表宽度改成70%"
"第4页图表的颜色改成绿色系"
```

### 7.4 批量调整

```
"所有页面的标题字号统一调大"
"把第3到第6页的背景都加深"
```

## 8. 场景 F：版式切换

**适用情况**：需要更换某页的布局模板（如从单列改为双栏）。

```
"把第7页换成双栏布局"
"第3页改成左图右文的版式"
"第5页用 stat-cards 展示这几个数字"
```

AI 步骤：
1. 读取 slide-tree.json，找到对应页面
2. 修改 layout.template 字段（如 `text-default` → `text-two-column`）
3. 重新运行 convert.py
4. 告知修改结果

可用版式见[第 12 节](#12-版式模板参考)。

## 9. 场景 G：导出 PDF

**适用情况**：需要将演示文稿分发为 PDF 文件。

```
你："帮我把演示文稿转成 PDF"
```

AI 调用 `html2pdf.py`，自动检测已安装的浏览器（Edge → Chrome → playwright Chromium）：

```
检测到系统浏览器：Microsoft Edge
正在转换 Q4-report.html → Q4-report.pdf...
转换完成：Q4-report.pdf（9 页，1280×720px）
```

### PDF 特性

- 每张幻灯片对应一页 PDF
- 图表完整渲染（JS 渲染完成后截图）
- 背景颜色和渐变完整保留
- 无黑边（自动修复 transform 导致的边框问题）

### 若系统无可用浏览器

安装 playwright Chromium（见[第 16 节](#16-安装与依赖)），或用浏览器手动导出：

Chrome/Edge → Ctrl+P → 另存为 PDF → 勾选"背景图形"。

## 10. 场景 H：批量截图预览

**适用情况**：想快速检查所有页面的视觉效果，无需逐页翻阅。

```
你："帮我截图预览所有页面"
```

```bash
python scripts/preview.py --input demo.html --temp
```

```bash
python scripts/preview.py --input demo.html --clean
```

> **注意**：截图需要 playwright 环境（见[第 16 节](#16-安装与依赖)）。

## 11. 演示文稿操作说明

演示文稿以 HTML 文件形式保存，用 Chrome / Edge / Firefox 打开即可演示。

### 键盘操作

| 按键 | 操作 |
|------|------|
| `→` 右方向键 | 下一页 |
| `←` 左方向键 | 上一页 |
| `空格` | 下一页 |
| `↓` 下方向键 | 下一页 |
| `↑` 上方向键 | 上一页 |
| `PageDown` | 下一页 |
| `PageUp` | 上一页 |
| `Enter` | 下一页 |

### 触摸操作（平板/手机）

| 手势 | 操作 |
|------|------|
| 向左滑动 | 下一页 |
| 向右滑动 | 上一页 |

### 页码与进度条

- 右下角显示当前页码，格式 `3 / 12`
- 页面底部有细线进度条，随翻页更新

### 全屏演示

浏览器按 `F11`（Windows）全屏，演示文稿自动缩放适配屏幕，保持宽高比不变形。

### 离线使用

默认 HTML 通过 CDN 加载 Chart.js 和 highlight.js，需要网络才能显示图表。

**生成离线版本**：

```bash
python scripts/convert.py --input demo.md --output demo.html --tree slide-tree.json --inline-assets
```

或告诉 AI：

```
"生成离线可用的版本"
```

## 12. 版式模板参考

共 27 种版式模板，按类别分组：

### 文字类

| 模板 | 适用场景 | 特点 |
|------|---------|------|
| `text-default` | 通用内容页 | 标题 + 正文/列表，最常用 |
| `text-hero` | 重要观点突出 | 大标题居中，少量副文字，冲击力强 |
| `text-two-column` | 4-8 条要点 | 均分为两列，空间利用率高 |
| `text-three-column` | 3 组并列内容 | 三列分栏，结构清晰 |
| `statement` | 核心陈述 | 单句话全幅展示，适合金句页 |

> **多组内容分列技巧**：
> - 用 `:::col` 语法在 MD 中显式分列（推荐，无需配置 tree）
> - 或在 slide-tree.json 中设置 `splitMode: "group"`，让 two/three-column 按组分列而非按条目均分

### 引用类

| 模板 | 适用场景 |
|------|---------|
| `quote-center` | 引用语居中展示，适合结语 |
| `quote-side` | 引用语 + 旁边说明文字 |

### 图片类

| 模板 | 适用场景 |
|------|---------|
| `image-right` | 文字/列表为主，图片占右侧 |
| `image-left` | 文字/列表为主，图片占左侧 |
| `image-top` | 图片在上，说明在下 |
| `image-bottom` | 说明在上，图片在下 |
| `image-fullbleed` | 图片铺满背景，文字叠加（结语首选） |
| `image-only` | 纯图片页，无文字 |

### 图表类

| 模板 | 适用场景 |
|------|---------|
| `chart-full` | 单图表全屏，趋势/数据为主角 |
| `chart-right` | 图表占右侧，左侧为文字说明 |
| `chart-left` | 图表占左侧，右侧为文字说明 |
| `chart-bottom` | 图表在下，上方为说明 |

### 数据展示类

| 模板 | 适用场景 | 特殊参数 |
|------|---------|---------|
| `stat-cards` | 3-6 个关键数字/KPI | `columns` 控制列数（默认单行） |
| `table-full` | 数据表格全屏展示 | 支持滚动 |

**stat-cards columns 参数**：

| 卡片数量 | 推荐 columns | 效果 |
|---------|-------------|------|
| 3张对称 | 默认（不设置） | 单行三列 |
| 4张对称 | `columns: 2` | 2×2 网格 |
| 5张 | `columns: 3` | 2+3 排列 |
| 6张 | `columns: 3` | 2×3 网格 |
| 内容差异大 | 不设置 | 自然撑开 |

### 时间线类

| 模板 | 适用场景 |
|------|---------|
| `timeline-vertical` | 垂直时间线，发展历程 |
| `timeline-horizontal` | 水平时间线，阶段/步骤 |

### 对比类

| 模板 | 适用场景 |
|------|---------|
| `compare-two` | 两组方案/选项对比 |
| `compare-three` | 三组方案/选项对比 |

### 卡片类

| 模板 | 适用场景 | 特殊参数 |
|------|---------|---------|
| `card-grid` | 特性/功能/场景卡片展示 | `columns` 控制列数 |

每个列表项渲染为一张卡片（加粗文字 = 卡片标题，其余 = 正文）。

### 封面/章节类

| 模板 | 适用场景 |
|------|---------|
| `cover-image-bg` | 封面：图片铺满背景 + 文字叠加 |
| `cover-split` | 封面：左右分割，一侧文字一侧图片 |
| `section-header` | 章节分隔页（自动按章节数切换变体） |

**section-header 自动切换**：

- **≤4 个章节**：居中大标题 + 底部进度点（简洁，章节少不需要导航）
- **≥5 个章节**：左侧当前章节标题 + 右侧完整目录，当前章节高亮（帮助听众定位）

章节数由脚本自动统计所有 section-header 页，无需手动配置。

### 选择指南

```
你有什么数据？
```

## 15. 常见问题

### Q1：生成的 HTML 没有图表，只有空白区域

**原因**：浏览器无法访问 CDN 加载 Chart.js。

**解决**：
1. 确保浏览器可以访问网络
2. 或生成离线版本："帮我生成离线可用的版本"

### Q2：图表中的数据不正确

```
你："第4页的图表数据有误，正确数据是：Q1=100, Q2=130, Q3=158, Q4=200"
```

或直接编辑 `data/` 目录下的 CSV 文件，再告知 AI 重新生成。

### Q3：AI 找不到我说的元素

描述更具体，指明页码和位置：

```
不够具体："把标题改大"
更好的说法："把第3页的标题改大"

不够具体："图表移到左边"
更好的说法："把第5页的折线图移到左边"

该页有两个列表时："第4页有两个列表，把第二个列表的字体颜色改成灰色"
```

### Q4：PDF 导出后图表没有显示

**原因**：使用了浏览器手动打印（Ctrl+P），Chart.js 可能未渲染完成。

**解决**：安装 playwright 后让 AI 导出，playwright 会等待 JS 渲染完成再截图。

### Q5：想一次修改多处样式

一次可以描述多个修改：

```
"帮我做以下修改：
1. 第1页标题改成白色加粗
2. 所有页面的列表行间距增大
3. 第4页的图表移到右边
4. 第7页背景改成深一点的颜色"
```

### Q6：修改内容后，之前调的样式还在吗

- **只修改页面内容，页码不变** → 该页样式**自动保留**
- **增加或删除了页面** → 页码后移/前移的页的样式丢失，操作前 AI 会告知
- **元素 ID 发生变化** → 该元素样式重置，AI 会指出哪些丢失

### Q7：两列布局只有一列内容正常显示

**原因**：页面有多组「小标题+列表」（p+ul 结构），默认只拆第一组的 li 条目。

**解决方案**：

```
"第3页有多组小节，请用 splitMode=group 的双栏版式"
```

或在 MD 中用 `:::col` 语法显式分列（更推荐，效果更可控）。

### Q8：演示文稿能在手机上用吗

可以，支持触摸滑动翻页，页面会自动缩放适配屏幕。

### Q9：Windows 下中文字符出现乱码

**原因**：AI 的文件写入工具在 Windows GBK 环境下可能腐蚀特定中文字符。

**解决**：建议用 VS Code 等文本编辑器直接编写或修改 MD 文件，修改完告诉 AI 重新生成 HTML。
