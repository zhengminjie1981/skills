## 1. 快速入门

### md2slides 能做什么

md2slides 是一个 Claude Code skill，帮助你将**原始材料**（报告、文章、数据、笔记）转换为可在浏览器中演示的 HTML 幻灯片，并支持导出为 PDF。

**核心能力一览**：

| 能力 | 说明 |
|------|------|
| 智能策划 | AI 分析材料，自动规划页面结构和内容分配 |
| 数据图表 | 自动识别数据并生成柱状图、折线图、饼图等 |
| 主题风格 | 5 套预设主题，支持 16:9 / 4:3 两种比例 |
| 键盘翻页 | 方向键和空格键控制翻页，右下角显示页码 |
| 精细调整 | 通过对话调整任意页面、任意元素的样式 |
| PDF 导出 | 每张幻灯片对应一页 PDF，保持比例 |

### 典型工作流

```
你提供材料
    ↓
AI 策划 MD 内容（你可以修改）
    ↓
确认后生成 HTML（选主题和比例）
    ↓
浏览器查看
    ↓
按需调整样式（场景 E）/ 修改内容（场景 D）
    ↓
导出 PDF（可选）
```

### 如何触发 md2slides

直接用自然语言告诉 Claude 你要做什么：

```
"帮我把这份报告做成演示文稿"
"把 report.md 转成幻灯片"
"这些数据需要做一个 PPT 展示"
"用深色主题生成 16:9 的演示文稿"
```

## 2. 使用场景 A：从原始材料生成演示文稿

**适用情况**：你有一份文章、报告或笔记，想直接做成幻灯片，不需要自己规划结构。

### 步骤 1：提供材料

将原始材料粘贴给 Claude，或告知文件路径：

```
你：帮我把下面这份产品季报做成演示文稿：

[粘贴材料内容，或]
"帮我把 D:/work/Q4-report.md 做成演示文稿"
```

Claude 会自动分析材料，规划演示结构。

### 步骤 2：查看并确认内容策划

Claude 会展示策划结果，例如：

```
已完成内容策划，共 9 页：

页码  类型    标题
1     封面    2024 Q4 产品季报
2     目录    目录
3     内容    季度概览
4     内容    用户增长
5     数据    月活跃用户趋势（包含折线图）
6     内容    核心功能迭代
7     数据    收入结构（包含柱状图）
8     内容    下季度计划
9     结语    谢谢 & 联系方式

涉及数据文件：
- data/Q4-report-s5-users.csv（已创建）
- data/Q4-report-s7-revenue.csv（已创建）

请确认内容，或告知需要调整的地方。
```

**此时你可以**：
- 直接说"好的，继续"→ 进入 HTML 生成
- 提出修改意见：

  ```
  "第6页内容太少了，拆成两页"
  "把目录页去掉"
  "在第8页之前加一页风险分析"
  "第5页改用柱状图"
  ```

Claude 会根据意见调整后重新展示，直到你满意。

### 步骤 3：选择宽高比

Claude 会询问宽高比：

```
请选择宽高比：
A. 16:9（推荐，适合现代显示器和投影仪）
B. 4:3（适合传统投影仪）
```

直接回复 A 或 B，或说"16:9"/"4:3"。

### 步骤 4：选择主题风格

Claude 会根据内容推荐主题，并说明理由：

```
根据季报的商务汇报风格，建议使用 professional-dark（深色背景，蓝白文字）。
是否采用，或有其他偏好？
```

你可以回复：
- "好的" → 使用推荐主题
- "用浅色的" → Claude 改为 professional-light
- "有没有更活泼的" → Claude 建议 creative-gradient
- 直接说主题名："用 minimal-clean"

> **快捷方式**：说"用默认配置"可跳过宽高比和主题询问，直接使用 16:9 + professional-dark。

### 步骤 5：查看生成结果

HTML 生成完成后，Claude 会告知：

```
HTML 已生成：Q4-report.html（9 页）
- 用浏览器直接打开即可演示
- 左右方向键 / 空格键翻页
- 右下角显示当前页码

生成的文件：
  Q4-report.html        （演示文稿）
  Q4-report-tree.json   （元素结构数据，用于后续调整）
  data/Q4-report-s5-users.csv
  data/Q4-report-s7-revenue.csv
```

用浏览器打开 `Q4-report.html` 即可开始使用。

## 3. 使用场景 B：从已有 MD 文件生成演示文稿

**适用情况**：你已经有一个 Markdown 文件，想将它转换为 HTML 幻灯片，并允许 AI 帮你优化分页和内容。

### 方式一：AI 辅助转换（推荐）

```
你："把 slides-draft.md 转成演示文稿，帮我优化一下分页"
```

AI 会读取 MD 文件，重新策划内容，走场景 A 的标准流程。

### 方式二：直接转换（保留原有分页）

```
你："把 slides-draft.md 直接转成 HTML，不要改内容"
```

AI 会按 MD 中已有的 `---` 分页符直接生成 HTML，只询问宽高比和主题。

### MD 文件格式要求

如果你的 MD 文件已有分页设计，请遵循以下格式（AI 生成的 MD 也是这种格式）：

```markdown
---
title: 演示文稿标题
ratio: 16:9
theme: professional-dark
---

<!-- Slide 1: 封面 -->
# 主标题
## 副标题

---

<!-- Slide 2: 内容页 -->
# 这一页的主题
- 要点一
- 要点二
- 要点三

---

<!-- Slide 3: 数据页 -->
# 用户增长趋势

<!-- chart
type: line
dataSource: data/my-file-s3-users.csv
xField: 月份
yField: 用户数(万)
position: right
width: 58%
-->

> **注意**：`dataSource` 字段指定数据文件路径。旧版本使用 `dataFile`，现已废弃但仍然兼容（会提示警告）。

- 说明文字一
- 说明文字二
```

**分页规则**：
- `---`（三个连字符单独一行）表示分页
- 每页第一个 `#` 标题是页面主标题
- `## 副标题` 通常用于封面页的副标题或作者信息

## 4. 使用场景 C：含数据图表的演示文稿

**适用情况**：材料中包含数据（表格、统计数字、对比数据），需要在演示文稿中以图表形式展示。

### 4.1 AI 自动识别数据

当你提供的材料包含数据时，AI 会自动：
1. 识别可视化的数据段落
2. 提取数据并创建 CSV 文件（保存在 `data/` 目录）
3. 在对应页面规划合适的图表类型

你不需要做任何额外操作，直接提供材料即可。

**AI 会将以下内容转为图表**：
- 包含 3 个及以上数值的比较数据
- 按时间排列的数据（月度、季度、年度）
- 明确的占比/分布数据

**以下内容不会转为图表**（保留为文字）：
- 单个孤立数字（"增长了 30%"）
- 以文字描述为主的表格

### 4.2 指定图表类型

在策划确认阶段，你可以要求修改图表类型：

```
你："第5页的图改成饼图"
你："用折线图展示用户趋势，柱状图展示收入对比"
你："第7页不要图表，改成数据表格"
```

**可用图表类型**：

| 你说的 | 对应图表 | 最适合 |
|--------|---------|--------|
| 柱状图 | bar | 分类对比、排名 |
| 条形图 | bar-horizontal | 标签较长的横向对比 |
| 折线图 | line | 趋势、时间变化 |
| 面积图 | area | 趋势+总量感受 |
| 饼图 | pie | 占比（分类≤6个） |
| 环形图 / 甜甜圈图 | donut | 占比+中间显示数值 |
| 散点图 | scatter | 分布、相关性 |
| 数据表格 | table | 需要精确查看数值 |

### 4.3 指定图表位置

在策划阶段或生成后，可以指定图表在页面中的位置：

| 位置说法 | 布局效果 |
|---------|---------|
| "图在右边，文字在左" | 左文右图，各占约一半 |
| "图在左边，文字在右" | 左图右文，各占约一半 |
| "只显示图表，不要文字" | 图表占满内容区 |
| "图在下方，文字在上" | 上文下图 |
| "图居中，上下有说明" | 居中小图，上下配文字 |

### 4.4 自定义数据

如果 AI 提取的数据有误，或你想手动提供数据，可以直接修改 `data/` 目录下的 CSV 文件：

```
你："第5页的数据有问题，我来提供正确数据"
```

然后编辑对应的 CSV 文件（格式为 UTF-8 编码，第一行为表头）：

```csv
季度,收入(万元)
Q1,120
Q2,145
Q3,168
Q4,203
```

修改 CSV 后告知 Claude 重新生成该页的图表即可。

### 4.5 Obsidian Dataview 格式（可选）

如果你使用 Obsidian 并希望数据与笔记库关联，可以要求使用 Dataview 格式存储数据：

```
你："数据用 Obsidian Dataview 格式保存"
```

AI 会将数据保存为包含 dataview 代码块的 `.md` 文件，放在 `data/` 目录。

## 5. 使用场景 D：内容变更

**适用情况**：HTML 已生成，需要修改演示文稿的**内容**（文字、列表、页面结构、图表类型）而非视觉样式。

> **内容变更 vs 样式调整**
>
> | 修改类型 | 走哪个场景 |
> |---------|------------|
> | 文字错误、要点措辞、标题 | 内容变更（本场景） |
> | 增加/删除/移动页面 | 内容变更（本场景） |
> | 图表类型、数据源字段 | 内容变更（本场景） |
> | 字体大小、颜色、间距 | 样式调整（场景 E） |
> | 图表位置、宽高百分比 | 样式调整（场景 E） |

### 6.1 修改文字内容

直接描述你要改什么：

```
"第3页第二条要点改成'月活用户超過 30 万，同比增长 45%'"
"第5页标题改为'核心功能迭代'"
"将第4页第一条要点中的'问题'改为'振战'"
```

AI 会定位到 MD 文件中对应页面，修改文字，然后重新生成 HTML。

### 6.2 增加 / 删除列表条目

```
"第3页再加一条要点：'自然流量占比提升至 38%'"
"把第5页第三条删掉"
"第2页目录再加一条'五、经验展望'"
```

### 6.3 增加 / 删除 / 移动页面

```
"在第5页后面加一页'风险分析'，包含三条要点：..."
"删除第2页（目录页）"
"把第6页和第7页交换位置"
```

> **注意**：增/删页面会导致其后所有页面的页码改变，这些页的手工样式调整将被重置为主题默认値。
> AI 在执行前会告知您影响范围，您可以确认后再继续。

### 6.4 修改图表类型或数据源

```
"第4页的柱状图改成折线图"         ← 修改图表类型
"第5页的图表换用 data/new-data.csv"   ← 修改数据源
"第4页的图表换成展示 Q1-Q4 准确数值的表格"  ← 修改图表类型
```

> 图表类型和数据源的变更属于**内容变更**，会先更新 MD 再重新生成。
> 图表的位置和大小属于**样式调整**（场景 E），直接修改 HTML，不需重新生成。

### 6.5 样式保留说明

内容变更重新生成时，之前的**样式调整会尽量保留**：

| 情况 | 样式结果 |
|------|----------|
| 页面内容修改，页码不变 | 该页手工样式自动保留 |
| 增加新页（需当前页之后的页码全部后移） | 后移页的手工样式丢失（AI 提前告知） |
| 删除页面（其后页码前移） | 后移页的手工样式丢失（AI 提前告知） |
| 修改内容但 ID 变化 | 该元素样式重置为主题默认値，AI 说明哪些丢失 |

## 6. 使用场景 E：样式调整

**适用情况**：HTML 已生成，需要修改某些页面或元素的外观（不改内容）。



### 6.1 调整语法

用自然语言描述你要改什么，Claude 会自动定位并修改：

```
格式："第 X 页的[元素描述][调整内容]"
```

**你不需要知道元素 ID**，用自然描述即可。

### 6.2 文字样式调整

**字体大小**：

```
"第1页的标题字再大一点"
"把所有页面的正文字体缩小一点"
"第3页的列表字号改成小一点"
```

**文字颜色**：

```
"第1页的副标题颜色改成白色"
"第4页的标题改成黄色"
"第2页的列表文字颜色浅一点"
```

支持中文颜色描述：白色、黑色、红色、蓝色、绿色、黄色、橙色、紫色、灰色、深灰等。
也支持 CSS 颜色值：`#FF6B6B`、`rgba(255,255,255,0.8)` 等。

**字体粗细**：

```
"第3页的标题加粗"
"第5页的列表字体细一点"
```

### 6.3 布局调整

**间距**：

```
"第2页的列表行间距大一点"
"标题和内容之间的间距减小"
```

**对齐**：

```
"封面的文字居中对齐"
"第3页的标题左对齐"
```

### 6.4 背景与配色

```
"第1页（封面）背景改成深蓝色"
"第5页背景加深一点"
"把所有页面的背景换成白色"（整体换主题时更合适）
```

### 6.5 图表调整

**图表类型**：

```
"第4页的图改成折线图"
"第6页的饼图改成环形图"
```

**图表位置和大小**：

```
"第4页的图表移到左边，文字在右"
"第5页的图表宽度改成70%"
"第4页的图表高度调大一点"
```

**图表配色**：

```
"第4页图表的颜色改成绿色系"
"第5页的柱状图用红橙黄三种颜色"
```

### 6.6 页面级操作

**重新生成某页**：

```
"第3页内容重新整理一下，太拥挤了"
"第7页的图表数据有误，用这个 CSV 重新生成"
```

**添加/删除页面**（需回到 MD 编辑阶段）：

```
"在第5页后面新加一页，内容是..."
"把第2页（目录）删掉"
```

### 6.7 批量调整

```
"所有页面的标题字号统一改成大一点"
"所有列表的行间距调大"
"把第3到第6页的背景都加深"
```

## 7. 使用场景 F：导出 PDF

**适用情况**：需要将演示文稿分发为 PDF 文件，每张幻灯片对应一页。

### 步骤

```
你："帮我把演示文稿转成 PDF"
或
你："把 Q4-report.html 导出为 PDF"
```

Claude 会调用 `html2pdf.py` 脚本，使用 Chromium 渲染后逐页导出：

```
正在转换 Q4-report.html → Q4-report.pdf...
✅ 转换完成：Q4-report.pdf（9 页，1280×720px）
```

> **注意**：首次使用 PDF 导出功能需要安装 playwright 环境（见[第 12 节](#12-安装与依赖)）。
> 如在中国大陆遇到 Chromium 下载失败，可设置镜像：
> ```bash
> # Windows
> set PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright/
> # Linux/Mac
> export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright/
> playwright install chromium
> ```

### PDF 特性

- 每张幻灯片对应一页 PDF
- PDF 页面尺寸与演示文稿宽高比一致（16:9 → 1280×720px，4:3 → 1024×768px）
- 图表完整渲染（不会缺失）
- 背景颜色和渐变完整保留

### 若 playwright 未安装

Claude 会提示备选方案：

```
playwright 未安装，可选以下方式：

方案 A：安装 playwright（推荐，图表完整）
  pip install playwright pypdf
  playwright install chromium

方案 B：浏览器手动导出
  1. 用 Chrome 打开 HTML 文件
  2. Ctrl+P 打印
  3. 打印机选"另存为 PDF"
  4. 勾选"背景图形"选项
  注：图表可能部分缺失
```

## 8. 演示文稿操作说明

演示文稿以 HTML 文件形式保存，用浏览器（Chrome、Edge、Firefox）打开即可演示。

### 键盘操作

| 按键 | 操作 |
|------|------|
| `→`（右方向键） | 下一页 |
| `←`（左方向键） | 上一页 |
| `空格` | 下一页 |
| `↓`（下方向键） | 下一页 |
| `↑`（上方向键） | 上一页 |
| `PageDown` | 下一页 |
| `PageUp` | 上一页 |
| `Enter` | 下一页 |

### 触摸操作（平板/手机）

| 手势 | 操作 |
|------|------|
| 向左滑动 | 下一页 |
| 向右滑动 | 上一页 |

### 页码

右下角显示当前页码，格式为 `当前页 / 总页数`，例如 `3 / 12`。

### 进度条

页面底部有细线进度条，随翻页更新，直观显示演示进度。

### 全屏演示

浏览器按 `F11`（Windows）或 `Cmd+Shift+F`（Mac）全屏，演示文稿会自动缩放适配屏幕，保持宽高比不变形。

### 离线使用

默认生成的 HTML 通过 CDN（bootcdn.cn）加载 Chart.js 和 highlight.js，**需要网络连接才能正常显示图表和代码高亮**。

**方式一：生成时内联资源（推荐）**

```bash
# 使用 --inline-assets 选项，自动下载并内联所有 JS/CSS
python scripts/convert.py --input demo.md --output demo.html --inline-assets
```

首次运行时会下载资源到 `assets/` 目录缓存，之后无需网络即可生成离线版。

**方式二：让 AI 生成离线版本**

```
你："生成离线可用的版本"
```

Claude 会在转换时自动添加 `--inline-assets` 参数，生成的 HTML 文件较大但完全离线可用。

## 10. 图表类型参考

### 选择指南

```
你有什么数据？
├── 随时间变化的数据（月/季/年）
│   ├── 想看趋势走势 → 折线图（line）
│   └── 想看趋势+量级 → 面积图（area）
├── 不同类别的数值对比
│   ├── 类别名称较短 → 柱状图（bar）
│   └── 类别名称较长 → 条形图（bar-horizontal）
├── 各部分占整体的比例
│   ├── 类别 ≤ 6 个 → 饼图（pie）
│   └── 想在中间显示总数 → 环形图（donut）
├── 两个变量的关系/分布 → 散点图（scatter）
└── 需要精确查看每个数值 → 数据表格（table）
```

### 各图表类型说明

#### 柱状图（bar）

竖向柱子，直观展示各类别之间的大小对比。

```
适合：季度收入对比、各部门人数、产品销量排名
告诉 AI："用柱状图" / "用条形图（竖的）"
```

#### 条形图（bar-horizontal）

横向条形，适合类别名称较长的场景。

```
适合：各城市数据、功能使用频率排名
告诉 AI："用横向条形图" / "类别名字很长，用横的"
```

#### 折线图（line）

连线展示数值随时间（或其他序列）的变化趋势。

```
适合：月活用户趋势、股价走势、温度变化
告诉 AI："用折线图" / "展示趋势变化"
```

#### 面积图（area）

折线图的填充版本，在强调趋势的同时展现量级感。

```
适合：累计收入增长、访问量变化
告诉 AI："用面积图" / "折线图加上填充"
```

#### 饼图（pie）

圆形，展示各部分占整体的比例，类别数量不超过 6 个时效果最佳。

```
适合：用户来源分布、市场份额、预算占比
告诉 AI："用饼图" / "展示占比"
```

#### 环形图（donut）

饼图的空心版本，中间可显示总数或核心数值。

```
适合：同饼图场景，中间需要突出展示某个数值
告诉 AI："用环形图" / "用甜甜圈图"
```

#### 数据表格（table）

不使用图形，直接以样式化表格展示数据，适合需要精确查阅数值的场景。

```
适合：财务数据对照表、规格参数对比
告诉 AI："用表格" / "不要图表，直接显示数据"
```

## 11. 常见问题

### Q1：生成的 HTML 没有图表，只有空白区域

**原因**：图表无法在没有网络的情况下加载 Chart.js（CDN）。

**解决方法**：
1. 确保浏览器可以访问网络
2. 或要求 AI 生成离线版本："帮我生成离线可用的版本"

### Q2：图表中的数据不正确

**解决方法**：

```
你："第4页的图表数据有误，正确数据是：Q1=100, Q2=130, Q3=158, Q4=200"
```

AI 会更新对应的 CSV 文件并重新生成该页图表。

或者你可以直接编辑 `data/` 目录下的 CSV 文件，然后告诉 AI 重新生成。

### Q3：调整时 AI 找不到我说的元素

**解决方法**：描述更具体，指明页码：

```
❌ "把标题改大" （哪一页？）
✅ "把第3页的标题改大"

❌ "图表移到左边" （哪一页？）
✅ "把第5页的折线图移到左边"
```

如有多个同类元素：

```
✅ "第4页有两个列表，把第二个列表的字体颜色改成灰色"
```

### Q4：PDF 导出后图表没有显示

**原因**：使用了浏览器手动打印（未安装 playwright）。

**解决方法**：安装 playwright 后重新导出：

```bash
pip install playwright pypdf
playwright install chromium
```

然后告诉 AI 重新导出 PDF。

### Q5：想修改多处样式，怎样更高效

一次可以描述多个修改：

```
"帮我做以下修改：
1. 第1页标题改成白色加粗
2. 所有页面的列表行间距增大
3. 第4页的图表移到右边
4. 第7页背景改成深一点的颜色"
```

AI 会逐一处理并汇报修改结果。

### Q6：能不能修改演示稿的 MD 文件后重新生成

可以。直接编辑 MD 文件后告诉 AI：

```
"我修改了 report.md，请重新生成 HTML"
```

AI 会重新运行转换脚本，**覆盖原来的 HTML 文件**。
已有的样式调整会尽量保留（按元素 ID 匹配），如需保留原 HTML 备份请自行操作。

### Q9：修改内容重新生成后，之前调的样式还在吗

视情况而定：

- **只修改页面内容，页码不变** → 该页样式**自动保留**
- **增加或删除了页面** → 页码后移/前移的页的样式丢失，操作前 AI 会事先告知
- **元素 ID 发生变化**（如页面第一个标题被删除）→ 该元素样式重置，AI 指出哪些丢失

总结：对不涉及页码变化的小幅内容修改，样式通常不丢失。

### Q7：能不能对单独的数据列调整颜色

可以，在调整阶段指定：

```
"第5页的多系列柱状图，第一组（订阅收入）改成蓝色，第二组（广告收入）改成橙色"
```

### Q8：演示文稿能在手机上用吗

可以，演示文稿支持触摸滑动翻页，在手机和平板上可以正常使用。页面会自动缩放适配屏幕。

## 12. 安装与依赖

### 基础依赖（MD → HTML 转换）

```bash
pip install python-frontmatter markdown
```

### PDF 导出依赖

```bash
pip install playwright pypdf
playwright install chromium
```

> **说明**：`playwright install chromium` 会下载约 200MB 的 Chromium 浏览器，只需执行一次。

```bash
# 检查基础依赖
python -c "import frontmatter, markdown; print('基础依赖 OK')"

# 检查 PDF 依赖
python -c "from playwright.sync_api import sync_playwright; print('playwright OK')"

```

- Python 3.9+
- 网络连接（用于加载 Chart.js / highlight.js CDN，默认使用 bootcdn.cn）

- 如需离线使用，请使用 `--inline-assets` 选项
