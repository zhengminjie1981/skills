# md2slides MD 内容编写规范

> 编写 MD 文件时参考本文档。

---

## 1. MD 文件结构与分页规则

**MD 文件整体结构**：

```markdown
---
title: 演示标题
ratio: "16:9"
theme: professional-dark
---

<!-- Slide 1: Cover -->
# 封面标题
## 副标题

---

<!-- Slide 2: 目录 -->
# 目录

- 第一章
- 第二章

---

<!-- Slide 3: 内容页 -->
# 页面标题

- 要点1
- 要点2
```

**分页规则（严格）**：

| 规则 | 说明 |
|------|------|
| **分页符** | 独立一行的 `---`，上下各有一空行 |
| **页面注释** | `<!-- Slide N: 页面类型 -->` 写在 `---` 之后，`# 标题` 之前 |
| **Front Matter** | 文件开头的 `---...---` 是元数据，不是分页符 |
| **`#` 不分页** | `#`（h1）是页内标题，不触发分页 |

> **常见误解**：`##`（h2）也不分页，所有页面只用 `---` 分隔。

**页面注释类型关键词**（影响 `data-type` 属性）：

| 注释关键词 | 对应类型 | 示例 |
|-----------|---------|------|
| `cover` / `封面` / `title` | `cover`（封面样式）| `<!-- Slide 1: Cover -->` |
| `outro` / `结语` / `ending` | `outro`（结语样式）| `<!-- Slide 8: 结语 -->` |
| 其他 | `content`（默认）| `<!-- Slide 2: 目录 -->` |

## 2. chart 块格式

图表数据通过 `<!-- chart -->` 块驱动，CSV 文件放在 `data/` 目录：

```markdown
<!-- chart
type: bar
csv: data/report-s7-users.csv
xField: month
yField: count
title: 月活用户趋势
-->
```

**参数说明**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `type` | 是 | bar / line / area / pie / donut / scatter |
| `csv` | 是 | 相对于 MD 文件的 CSV 路径 |
| `xField` | 是 | X 轴字段名（CSV 列名） |
| `yField` | 是 | Y 轴字段名，多字段用逗号分隔 |
| `title` | 否 | 图表标题 |

⚠️ **省略 xField/yField 时的症状**：
- 不会报错，但图表渲染为空白（Y轴 0~1，无数据点）
- 必须显式指定 CSV 列名，convert.py 不会自动推断字段

**图表类型选择**：

```
时间序列 / 趋势  ->  line 或 area
分类对比（8类以内）->  bar
分类对比（标签长）->  bar-horizontal
占比（6类以内）  ->  pie 或 donut
多维分布         ->  scatter
精确数值参考     ->  table（原生 MD 表格，配合 table-full 版式）
```

## 3. splitMode:group 的前提

`splitMode: group` 要求 MD 页面内有多组**标题/段落 + 列表**交替结构：

```markdown
# 方式1：粗体段落 + 列表（推荐）
**组标题A**

- 列表项1
- 列表项2

**组标题B**

- 列表项3
- 列表项4

# 方式2：## 标题 + 列表（已支持）
## 组标题A

- 列表项1
- 列表项2

## 组标题B

- 列表项3
- 列表项4
```

**支持的组合**：`<p>` + `<ul>`、`<h1>` + `<ul>`、`<h2>` + `<ul>`、`<h3>` + `<ul>`

⚠️ **空行是强制要求，不是风格偏好**：

| 写法 | Markdown 解析结果 | splitMode:group |
|------|-----------------|----------------|
| `**标题**\n- 列表项`（无空行）| 合并为单个 `<p>`，列表消失 | ❌ 找不到组，回退 |
| `**标题**\n\n- 列表项`（有空行）| `<p>` + `<ul>` 正确配对 | ✅ 正常分组 |

- `**粗体**` 是行内元素，没有空行时 Markdown 将其与下一行合并成同一个 `<p>`
- `#`/`##`/`###` 是块级元素，对空行不敏感，但建议统一加空行保持一致性

**不满足条件时自动回退**（convert.py 会打印 WARNING）：
- 只有一个列表 → 触发回退，按 item split 处理
- 无列表结构 → 降级为默认布局

## 4. stat-cards / card-grid 内容规范

### stat-cards 布局

**强制要求**：内容必须是 `<li>` 列表项，数字会被 `_wrap_key_numbers` 自动提取并放大

```markdown
# 正确写法

- **140 万亿元** GDP 总量
- **2.8%** 研发投入强度
- **35 万** 月活用户

❌ 错误写法（不会生成卡片）

GDP 突破 140 万亿元    ← 这是 <p>，不是 <li>
```

**每张卡片内容必须极简**：**加粗数字 + 不超过一行说明**

**卡片数量与 columns 参数**：
- 3 项 → 默认单行（无需设置 columns）
- 4 项 → `columns: 2`（2×2）
- 5-6 项 → `columns: 3`（2×3 或 2+3）

### card-grid 布局

**强制要求**：每个 `<li>` 内的 `<strong>` 作为卡片标题

```markdown
# 正确写法

- **算力基础设施** 描述文字会自动成为卡片正文
- **模型算法** 支持多行描述
- **应用场景** 第三个卡片

❌ 错误写法（无标题，卡片视觉弱）

- 算力基础设施：描述文字    ← 无 <strong>，标题为空
```

**适用场景**：4-6 个独立完整单元（功能、特性、场景）

**不适用场景**：
- 4 条以内"关键词：说明"格式 → 改用 text-default + 列表项内 **关键词** 加粗
- 长段落描述（>30字/项）→ 卡片溢出，改用 text-default

## 5. 引用块（导语）语法

在分栏布局（text-two-column/text-three-column）中，引用块 `>` 会渲染为 `<blockquote>`，独占全宽，不会被纳入分栏内容。

**适用场景**：分栏前的导语、背景说明、核心观点

```markdown
# 标题

> 导语独占全宽，不会被纳入分栏

**第一组**

- 项目1
- 项目2

**第二组**

- 项目3
```

**渲染结果**：
- `<blockquote>` 自动成为分栏前的独立前缀
- 后续的 `p+ul` 分组正常进入左右两列

**不使用引用块**：
```markdown
# 标题

导语文字（这是 <p>，会被纳入第一列）

**第一组**

- 项目1
```

## 6. Emoji 使用规范

### 适用范围

| 页面类型 | 是否使用 emoji |
|---------|--------------|
| content 页列表项 | 推荐（每页 40-60% 的条目） |
| toc 目录条目 | 可选（风格一致即可） |
| 标题（h1/h2） | 不建议 |
| 封面页、结语页 | 不建议 |
| data 页（图表+文字说明） | 少量，不超过 2 个 |

### 关键词 → emoji 映射

| 关键词（含义）| 推荐 emoji |
|------------|-----------|
| 增长/提升/突破 | 📈 |
| 下降/减少/降低 | 📉 |
| 用户/客户/团队 | 👥 |
| 收入/利润/营收 | 💰 |
| 目标/计划/OKR | 🎯 |
| 风险/问题/挑战 | ⚠️ |
| 技术/功能/特性 | ⚙️ |
| 数据/分析/报告 | 📊 |
| 时间/进度/排期 | 📅 |
| 新功能/发布/上线 | 🚀 |
| 合作/整合/协作 | 🤝 |
| 安全/隐私/防护 | 🔒 |
| 全球/扩张/出海 | 🌏 |
| 第一/最优/最佳 | 🏆 |
| 创新/创意/想法 | 💡 |

### 使用原则

1. 每页不超过 60% 的列表项加 emoji
2. 同一页风格一致，不混用
3. emoji 位于条目开头：`- 📈 月活突破 35 万`
4. 用户明确要求"不加 emoji"或"正式风格"时不使用

## 6. 文件写入规范（Windows 中文）

**根因**：Write/Edit 工具在 Windows GBK 环境下会腐蚀特定中文字符（与 GBK 编码冲突的 Unicode 字符）。

**规范**：
- 含中文的 MD/JSON 文件用 Python `open(..., 'w', encoding='utf-8')` 写入
- 避免用 Write/Edit 工具直接写大段中文内容
- 验证：`python -c "import re; t=open('f.md',encoding='utf-8').read(); print(len(re.findall(chr(0xfffd),t)), '个乱码')"`
