# 对话式填写模板

读取用户提供的模板，分段对话收集信息，逐段展示草稿确认，最终保存完整文档。

## 核心流程

1. **读取模板** - 读取用户指定的模板文件路径
2. **分析结构** - 识别所有 Markdown 章节（H2/H3），制定问题清单
3. **分段访谈** - 逐章节：提问（1-3 个）→ 收集回答
4. **分段预览** - 展示该章节草稿 → 用户确认或修改
5. **保存文件** - 所有章节完成后，生成完整文档并写入文件

## 访谈协议

- 每次只处理一个章节
- 每个章节提问不超过 3 个（复杂章节可追问）
- 每个章节草稿展示后等待确认，"OK/好/继续" 才进入下一章节
- 用户说"跳过"则跳过该章节
- 用户说"返回"则重新访谈上一个章节
- 所有章节完成后，询问保存路径

## 使用示例

```
/template-filler /path/to/template.md
/template-filler ./templates/report.md -o ./output/report.md
```

## 典型交互

```
用户: /template-filler /path/to/template.md
AI: 已读取模板，共 5 个章节：[列出章节名]
    我们从第一章节「项目背景」开始。
    问题1：这个项目要解决什么核心问题？

用户: 是一个...
AI: [展示「项目背景」草稿]
    ---
    ## 项目背景
    （生成的内容）
    ---
    这样描述准确吗？说"继续"进入下一章节。
```
