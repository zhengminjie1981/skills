# Skill 设计模式

> 通用的 skill 设计模式和最佳实践。

---

## 模式 1: YAML Front Matter 设计

### 标准格式

```yaml
---
name: skill-name
description: |
  简短描述（~50 字）。

  **触发场景**（包含以下关键词）：
  - 场景1：关键词A、关键词B
  - 场景2：关键词C、关键词D

  **不触发场景**：
  - 场景X：关键词Y
---
```

### 设计要点

1. **触发场景**：
   - 使用 Markdown 加粗标签
   - 场景分组，便于理解
   - 关键词具体，避免泛化

2. **不触发场景**：
   - 明确边界，避免误触发
   - 列出容易混淆的场景

3. **description 是 AI 判断的唯一依据**：
   - 必须完整描述触发条件
   - 不要在 body 中补充触发条件

## 模式 2: SKILL.md 结构

### 推荐结构

```
1. 一行总结
2. 快速开始（引用 scripts/ 中的命令）
3. 核心功能表格
4. 决策树/工作流程
5. 典型场景示例（2-3 个）
6. 常见问题/故障排除
7. 参考资源索引
```

### 各部分要点

| 部分 | 要点 |
|------|------|
| 一行总结 | 让用户 5 秒内理解 skill 做什么 |
| 快速开始 | 命令可直接复制执行 |
| 核心功能表格 | 功能、说明、触发示例 |
| 决策树 | ≤ 5 步，二选一分支 |
| 场景示例 | 对话形式或命令形式 |
| 参考资源 | 任务→文档映射表 |

## 模式 3: 决策树设计

### 简单决策（推荐）

```
问题1？
├─ YES → 路径A
│         └─ 下一步
└─ NO  → 路径B
          └─ 下一步
```

### 复杂流程

```
1. 步骤1 → 说明
2. 步骤2（如需）
3. 步骤3 → 检查点
4. 步骤4
```

**原则**：
- 简单决策用二叉树
- 复杂流程用线性步骤
- 关键节点设置检查点

## 模式 4: 按需加载设计

### 加载层级

| 层级 | 内容 | 加载时机 |
|------|------|---------|
| 1 | Front matter | 总是加载 |
| 2 | SKILL.md body | skill 触发时 |
| 3 | references/*.md | AI 按需读取 |
| 4 | scripts/, docs/, data/ | **永不加载** |

### references/ 组织方式

**简单 skill**：
```
references/
├── spec.md           # 完整规范
├── troubleshooting.md
└── examples.md
```

**复杂 skill**：
```
references/
├── core/             # 核心规范
│   ├── spec.md
│   └── workflow.md
├── execution/        # 执行支持
│   ├── tools.md
│   └── checklist.md
└── advanced/         # 高级主题
    └── config.md
```

### 按需加载指引

```markdown
## 参考资源

| 任务 / 场景 | 读取文档 | 关键内容 |
|------------|---------|---------|
| 场景A | `references/xxx.md` | 说明 |
| 场景B | `references/yyy.md` | 说明 |
```

## 模式 5: 目录结构设计

### 标准结构

```
skill-name/
├── SKILL.md              # 入口（AI 触发时加载）
├── references/           # AI 参考文档（按需加载）
├── scripts/              # 可执行代码（不会被加载）
├── docs/                 # 开发文档（不会被加载）
├── examples/             # 示例文件（不会被加载）
├── tests/                # 测试文件（不会被加载）
├── data/                 # 运行时数据（不会被加载）
└── assets/               # 输出资源（不会被加载）
```

### 目录用途

| 目录 | AI 加载 | 内容类型 |
|------|---------|---------|
| `references/` | ✅ 按需 | .md 文档、配置模板 |
| `scripts/` | ❌ | 可执行代码 (.py, .sh) |
| `docs/` | ❌ | 设计、变更日志 |
| `examples/` | ❌ | 示例输入/输出 |
| `tests/` | ❌ | 测试用例 |
| `data/` | ❌ | 运行时配置、缓存 |
| `assets/` | ❌ | 模板、静态资源 |

### 代码存放规则

| 代码类型 | 存放位置 |
|---------|---------|
| 实际执行代码 | `scripts/*.py` |
| 示例代码（展示用法） | Markdown 代码块 |
| 配置模板 | `references/*.txt` |
| 运行时配置 | `data/*.yaml` |

## 模式 6: 场景示例设计

### 对话形式

```
用户: {{用户输入}}
AI: ✓ 步骤1 → 步骤2 → 步骤3
    ✓ 结果说明
```

### 命令形式

```markdown
### 场景: {{场景名称}}

\`\`\`bash
python scripts/tool.py [options]
\`\`\`

**Output:** 预期输出说明
```

### 问题解决形式

```markdown
### 场景: {{问题描述}}

用户: {{问题描述}}
→ 步骤1: 说明
→ 步骤2: 说明
```

## 模式 7: AI Agent 优化

### 设计原则

**简洁即 AI 友好**：精简的文档减少 token 消耗，清晰的结构降低理解成本。

| 做法 | AI 友好原因 |
|------|------------|
| 表格 > 长段落 | 结构化信息，易于解析 |
| 决策树 | 明确分支，减少判断 |
| 默认值 | 减少选择，快速行动 |
| 任务→文档映射 | 按需加载，精准定位 |
| 标记特殊情况 | 明确边界，避免错误 |

## 总结：最佳实践

### Front Matter

1. 使用 Markdown 格式
2. 明确触发场景 + 关键词
3. 明确不触发场景

### SKILL.md 结构

1. 一行总结
2. 快速开始（引用 scripts/）
3. 决策树（≤ 5 步）
4. 核心功能表格
5. 场景示例（2-3 个）
6. 参考资源索引

### 目录结构

1. `references/` 仅存放 .md 文档
2. 实际执行代码放在 `scripts/`
3. 开发文档放在 `docs/`
4. Markdown 中只有示例代码

### 按需加载

1. 使用表格指引（任务→文档）
2. references/ 按主题组织
3. SKILL.md < 500 行

### AI 优化

**简洁即 AI 友好**：
1. 表格 > 长段落
2. 决策树 + 默认值
3. 任务→文档映射表
4. 标记特殊情况
