# 实现阶段检查清单

> 在发布之前，确保 skill 符合以下标准。

## 目录结构

- [ ] 目录结构符合规范（见 SKILL-GUIDE.md）
- [ ] `references/` 仅包含 `.md` 文档和配置模板
- [ ] 实际执行代码放在 `scripts/` 目录
- [ ] 开发文档放在 `docs/` 目录
- [ ] 示例文件放在 `examples/` 目录
- [ ] 测试文件放在 `tests/` 目录
- [ ] 运行时数据放在 `data/` 目录

## YAML Front Matter

- [ ] `name` 字段存在且为小写字母和连字符
- [ ] `description` 包含功能描述
- [ ] `description` 包含触发场景和关键词
- [ ] `description` 包含不触发场景（如有必要）
- [ ] 没有多余的 YAML 字段

## SKILL.md 内容

### 结构完整性

- [ ] 有一行总结（让用户快速理解）
- [ ] 有快速开始（引用 scripts/ 中的命令）
- [ ] 有核心功能表格
- [ ] 有典型场景示例（2-3 个）
- [ ] 有常见问题（如有）
- [ ] 有参考资源索引

### 快速开始

- [ ] 命令引用 `scripts/` 中的脚本
- [ ] 包含必要的参数
- [ ] 有预期输出说明

### 决策树/流程图

- [ ] 决策步骤 ≤ 5 步
- [ ] 每个分支有明确的条件
- [ ] 结果指向具体操作

### 参考资源

- [ ] 按主题组织 references/
- [ ] 每个文档有明确用途
- [ ] 在 SKILL.md 中说明何时加载哪个文档

## 代码存放规范

- [ ] 实际执行代码在 `scripts/*.py` 或 `scripts/*.sh`
- [ ] Markdown 文件中只有示例代码（代码块形式）
- [ ] `references/` 中没有 `.py` 或 `.sh` 可执行文件
- [ ] 配置模板（如 requirements.txt）可放在 `references/`

## 脚本 (scripts/)

- [ ] 脚本有明确的用途
- [ ] 脚本有帮助信息（--help）
- [ ] 脚本有错误处理
- [ ] 脚本路径在 SKILL.md 中说明

## 文档 (references/)

- [ ] 文件名清晰表达内容
- [ ] 文档有标题和目录（如需要）
- [ ] 内容不与 SKILL.md 重复
- [ ] 文档之间有清晰的关联
- [ ] 仅包含 `.md` 文件和配置模板

## 开发文档 (docs/)

- [ ] 设计文档（如有）放在 `docs/`
- [ ] 变更日志（如有）放在 `docs/`
- [ ] 归档文档放在 `docs/_archive/`

## 格式规范

- [ ] 使用 UTF-8 编码
- [ ] 使用 LF 换行符
- [ ] Markdown 格式正确
- [ ] 表格对齐
- [ ] 代码块有语言标识

## 长度控制

- [ ] SKILL.md < 500 行
- [ ] 单个 reference 文件 < 1000 行
- [ ] 如超过，考虑拆分

## 自检清单

| 检查项 | 状态 |

| 按需加载的指引清晰吗？ | |
